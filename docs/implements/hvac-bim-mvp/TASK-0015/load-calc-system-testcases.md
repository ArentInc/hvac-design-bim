## 2. 異常系テストケース（エラーハンドリング）

### TC-E01: 必要風量 -- ゼロ除算防止（supplyAirTempDiff = 0）

- **テスト名**: 送風温度差が 0 の場合にゼロ除算せず 0 を返す
  - **エラーケースの概要**: supplyAirTempDiff = 0 はゼロ除算を引き起こす
  - **エラー処理の重要性**: ゼロ除算は Infinity や NaN を生み、後続計算を破壊する
- **入力値**: `coolingLoad=15000`, `supplyAirTempDiff=0`
  - **不正な理由**: 送風温度差 0 は物理的にありえない条件
  - **実際の発生シナリオ**: 設計条件の入力ミスや初期化漏れ
- **期待される結果**: `0` (m^3/h)
  - **エラーメッセージの内容**: エラーメッセージなし（0 を返すことで安全に処理）
  - **システムの安全性**: NaN/Infinity を防ぎ、後続計算に影響しない
- **テストの目的**: ゼロ除算防止ロジックの確認
  - **品質保証の観点**: NFR-102（計算失敗時のクラッシュ防止）
- 🔵 TASK-0015 テスト6 に明示
