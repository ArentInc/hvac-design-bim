## 1. 正常系テストケース（基本的な動作）

### TC-N01: 交差検出 -- 完全一致する外壁面のセグメント検出

- **テスト名**: 交差検出 -- 完全一致する外壁面のセグメント検出
  - **何をテストするか**: ゾーン境界の辺と外壁面が完全に一致する場合、正しい perimeterSegment が返却されること
  - **期待される動作**: 10m 幅の南面外壁がゾーン南辺と完全一致し、orientation='S' で wallArea=10*wallHeight のセグメントが生成される
- **入力値**:
  - zoneBoundary: `[[0,0],[10,0],[10,10],[0,10]]`（10m x 10m 正方形）
  - architectureWalls: `[{ wallId: 'w1', orientation: 'S', wallArea: 27, glazingRatio: 0.4, vertices: [{x:0,y:0,z:0},{x:10,y:0,z:0}] }]`
  - wallHeight: 2.7
  - **入力データの意味**: 最も基本的なケースとして、ゾーン南辺と外壁面が完全に重なる単純な正方形ゾーンを使用
- **期待される結果**: `[{ orientation: 'S', wallArea: 27, glazingRatio: 0.4 }]`（セグメント 1 件、wallArea = 10m x 2.7m = 27）
  - **期待結果の理由**: 交差長 10m x 壁高さ 2.7m = 27m2。建築参照の glazingRatio をそのまま引き継ぐ
- **テストの目的**: 最も基本的な交差検出パターンの正常動作を確認
  - **確認ポイント**: 返却されるセグメント数、orientation、wallArea、glazingRatio の正確性
- 🔵 **青信号**: REQ-208、TASK-0014 テスト1 に明示的記載

### TC-B05: wallArea = 0（壁面積ゼロ）

- **入力値**: `{ orientation: 'S', wallArea: 0, glazingRatio: 0.3 }`

  - **境界での正確性**: 0 は現行スキーマで有効値
  - **一貫した動作**: 壁面積 0 でも保存可能

### TC-B06: 交差検出 -- 交差長がイプシロン以下の場合

  - **境界値での動作保証**: 微小な交差は実質的に交差なしとして扱われる

  - **境界値選択の根拠**: TASK-0014 注意事項のイプシロン 0.001m
  - **実際の使用場面**: モデリング時の微小な座標ずれ
- **期待される結果**: `[]`（空配列）

  - **一貫した動作**: 0.001m 以下は非交差、0.001m 超は交差として扱う

  - **堅牢性の確認**: 浮動小数点演算の誤差に対する耐性
- 🟡 **黄信号**: TASK-0014 注意事項「イプシロン 0.001m」に基づく妥当な推測

### TC-B07: 交差検出 -- 空の architectureWalls
