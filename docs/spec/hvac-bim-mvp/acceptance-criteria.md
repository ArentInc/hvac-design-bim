## REQ-001〜REQ-010: 基盤整備 🔵

**信頼性**: 🔵 *PRDセクション14・CLAUDE.md・ヒアリングQ2,Q5より*

### Given（前提条件）
- Pascal Editor のフォークリポジトリが存在する
- 既存の14ノードタイプが動作している

### When（実行条件）
- HVAC ノードスキーマを追加し、AnyNode 判別共用体を拡張する
- 建築/HVAC 2モード制を実装する

### Then（期待結果）
- 7種のHVACノード（HvacZone, System, AHU, Diffuser, DuctSegment, DuctFitting, PipeSegment）が AnyNode に含まれる
- 各ノードの Zod スキーマが parse/safeParse で正しく動作する
- 「建築モード」と「HVACモード」が上部バーから切替可能
- HVACモードで zone/equip/route/calc の4フェーズが表示される

### テストケース

#### 正常系

- [ ] **TC-001-01**: HvacZoneNode スキーマが全必須項目で parse 成功する 🔵
  - **入力**: zoneName, usage, floorArea, ceilingHeight, designConditions, perimeterSegments, boundary を含むオブジェクト
  - **期待結果**: Zod parse が成功し、型付きオブジェクトが返る

- [ ] **TC-001-02**: SystemNode スキーマが全必須項目で parse 成功する 🔵
  - **入力**: systemName, servedZoneIds, ahuId, aggregatedLoad, status を含むオブジェクト
  - **期待結果**: Zod parse が成功する

- [ ] **TC-001-03**: AhuNode スキーマが全必須項目で parse 成功する 🔵
  - **入力**: tag, equipmentName, position, ports, airflowRate, coolingCapacity, heatingCapacity, staticPressure, systemId を含むオブジェクト
  - **期待結果**: Zod parse が成功する

- [ ] **TC-001-04**: DuctSegmentNode スキーマが ductMaterial を含んで parse 成功する 🔵
  - **入力**: start, end, medium, shape, ductMaterial(`galvanized`), airflowRate, systemId を含むオブジェクト
  - **期待結果**: Zod parse が成功する

- [ ] **TC-001-05**: PipeSegmentNode スキーマが medium 区別で parse 成功する 🔵
  - **入力**: medium を `chilled_water_supply` と `chilled_water_return` で各1件
  - **期待結果**: 両方で Zod parse が成功する

- [ ] **TC-001-06**: モード切替で建築→HVAC に遷移する 🔵
  - **入力**: 建築モードでモード切替ボタンクリック
  - **期待結果**: フェーズが zone に切り替わり、HVACツールが有効化される

- [ ] **TC-001-07**: モード切替でHVAC→建築 に遷移する 🔵
  - **入力**: HVACモードでモード切替ボタンクリック
  - **期待結果**: フェーズが site に切り替わり、建築ツールが有効化される

#### 異常系

- [ ] **TC-001-E01**: 必須項目欠落時に parse が失敗する 🔵
  - **入力**: HvacZoneNode で zoneName を省略
  - **期待結果**: Zod parse エラーが返る

- [ ] **TC-001-E02**: 不正な nodeType でAnyNode parse が失敗する 🟡
  - **入力**: type が `invalid_type` のオブジェクト
  - **期待結果**: discriminatedUnion の parse エラーが返る

## REQ-301〜REQ-306: 負荷概算 🔵

**信頼性**: 🔵 *PRDセクション11.4・15.1・20.8より*

### Given（前提条件）
- HvacZone が作成済み、用途・面積・外皮条件が設定済み

### When（実行条件）
- 負荷計算を実行する

### Then（期待結果）
- coolingLoad, heatingLoad, requiredAirflow が算出される
- 方位別外皮補正内訳（perimeterLoadBreakdown）が含まれる

### テストケース

#### 正常系

- [ ] **TC-301-01**: 内部負荷のみのゾーンで冷房負荷が正しく算出される 🔵
  - **入力**: office_general, 120m², ペリメータなし
  - **期待結果**: coolingLoad = 120 × 150 = 18,000W (18kW)

- [ ] **TC-301-02**: ペリメータゾーンで外皮補正が適用される 🔵
  - **入力**: office_general, 120m², S面(wallArea=10.8, glazingRatio=0.7)
  - **期待結果**: 内部負荷 18kW + 外皮負荷（S面日射補正 × ガラス + 貫流）

- [ ] **TC-301-03**: 必要風量が正しく算出される 🔵
  - **入力**: coolingLoad=25,200W, supplyAirTempDiff=10
  - **期待結果**: requiredAirflow = 25200 / (1.2 × 1005 × 10) ≈ 2.09 m³/s ≈ 7,524 m³/h

- [ ] **TC-301-04**: 方位別日射補正係数が正しく適用される 🔵
  - **入力**: 各方位のペリメータセグメント
  - **期待結果**: S=1.0, E/W=1.2, N=0.6 等が適用される

- [ ] **TC-301-05**: perimeterLoadBreakdown に方位別内訳が含まれる 🔵
  - **入力**: S面とE面のペリメータ
  - **期待結果**: S面補正値とE面補正値が個別に記録される

- [ ] **TC-301-06**: 会議室の原単位が正しく参照される 🔵
  - **入力**: usage=conference, 40m²
  - **期待結果**: coolingLoad = 40 × 200 = 8,000W (8kW)

#### 境界値

- [ ] **TC-301-B01**: ペリメータなしゾーンで外皮補正が0になる 🔵
  - **入力**: perimeterSegments が空配列
  - **期待結果**: 外皮負荷 = 0、内部負荷のみ
