# hvac-bim-mvp 準備タスク（ユーザー作業）

> **仕様**: [requirements.md](requirements.md)
> **生成日**: 2026-03-26

**【信頼性レベル凡例】**:
- 🔵 **青信号**: 要件定義書・PRD・ユーザヒアリングで明確に必要と判明したタスク
- 🟡 **黄信号**: 要件定義書・PRDから妥当に推測されるタスク
- 🔴 **赤信号**: 推測による予防的タスク（実装時に不要と判明する可能性あり）

## 必須（実装開始前に完了が必要）

以下のタスクが完了していないと、実装フェーズでブロッカーになります。

- [ ] **サンプルデータセット JSON の作成** 🔵 *PRDセクション20より*
  - PRDセクション20で定義された以下の6ファイルを `packages/core/src/data/` に作成する:
    - `sample-architecture.json`（建築参照モデル）
    - `catalog-ahu.json`（AHU カタログ 5機種）
    - `catalog-diffuser.json`（制気口カタログ 6サイズ）
    - `standard-duct-sizes.json`（ダクト標準サイズ表）
    - `standard-pipe-sizes.json`（配管標準口径表）
    - `load-unit-table.json`（負荷原単位テーブル 5用途）
  - JSON スキーマは PRD に定義済み。そのまま転記可能
  - 関連要件: REQ-004, REQ-305, REQ-505, REQ-603

- [ ] **負荷計算の手計算検証データ準備** 🔵 *PRDセクション23.3より*
  - デモシナリオ（3ゾーン事務所ビル）の手計算結果を事前に用意する
  - Z-01 南側事務室: 冷房負荷≈25kW, 暖房負荷≈12kW, 風量≈7,500m³/h
  - Z-02 北側事務室: 冷房負荷≈30kW, 暖房負荷≈18kW, 風量≈9,000m³/h
  - Z-03 会議室: 冷房負荷≈10kW, 暖房負荷≈5kW, 風量≈3,000m³/h
  - ±20%以内の一致を受け入れ基準とする
  - 関連要件: NFR-301

## 推奨（実装中に用意できればOK）

実装を開始できますが、該当機能の実装前までに準備してください。

- [ ] **ダクト圧損の手計算検証データ** 🔵 *PRDセクション23.3より*
  - デモシナリオのダクト系統（幹線+3分岐+フレキ）の手計算結果
  - ±15%以内の一致を受け入れ基準とする
  - 必要になるフェーズ: MVP-3
  - 関連要件: NFR-302

- [ ] **プリセットデータ 5種の作成** 🔵 *PRDセクション21.4より*
  - preset-00-empty 〜 preset-04-complete の5段階
  - 各プリセットはデモシナリオの途中状態を JSON で保存したもの
  - 実装の進行に合わせて段階的に作成可能
  - 必要になるフェーズ: MVP-4
  - 関連要件: REQ-1701

## 確認事項（判断が必要）

実装方針に影響するため、早めの判断・確認が推奨されます。

- [ ] **DuctFittingNode の詳細仕様** 🟡 *PRDセクション14.1で言及あるが詳細なし*
  - PRD ではエルボ・T分岐の圧損係数テーブルは定義されているが、DuctFittingNode 自体のスキーマ（ノード属性、接続ポート構造）の詳細が薄い
  - 設計フェーズで仕様を確定する必要あり
  - 関連要件: REQ-705

- [ ] **HVACモード←→建築モード切替時のUI詳細** 🟡 *ヒアリングQ5で方針は確定、UI詳細は未定義*
  - 切替ボタンの配置場所（上部バーのどこか）
  - 切替時のアニメーション・遷移
  - 建築モードでのHVACノード表示/非表示
  - 設計フェーズで確定予定
  - 関連要件: REQ-1401

- [ ] **既存 ZoneNode と HvacZoneNode の参照連携** 🟡 *ヒアリングQ2で分離は確定、連携は未定義*
  - 建築用 ZoneNode の境界をHvacZoneNodeの初期境界として参照できるかの判断
  - 完全に独立でもMVPとしては成立する
  - 関連要件: REQ-201
