## エピック1: 基盤整備とモード切替

### ストーリー 1.1: HVACモードへの切替 🔵

**信頼性**: 🔵 *PRDセクション13・ヒアリングQ5より*

**私は** 空調設備設計者 **として**
**建築モードからHVACモードに切り替えたい**
**そうすることで** 建築図面を参照しながら空調設計作業を開始できる

**関連要件**: REQ-002, REQ-006, REQ-1401

**詳細シナリオ**:
1. ユーザーは上部バーのモード切替ボタンをクリックする
2. 画面が「HVACモード」に切り替わり、Zone / Equip / Route / Calc のフェーズタブが表示される
3. 建築要素（Wall/Slab等）は参照表示（編集不可）として残る
4. HVACツールが有効化される

**前提条件**:
- 建築モードで建物が作成済みである（またはサンプル建築が読込済み）

**制約事項**:
- モード切替は既存の建築データを変更しない
- HVACモードでは建築要素の編集ツールは無効

**優先度**: Must Have

---

### ストーリー 1.2: サンプルデータの読込 🔵

**信頼性**: 🔵 *PRDセクション20・ヒアリングQ6より*

**私は** 空調設備設計者 **として**
**サンプル建築参照データを読み込みたい**
**そうすることで** 既存の建築データなしでもHVAC設計のワンパスを試せる

**関連要件**: REQ-102, REQ-103, REQ-004

**詳細シナリオ**:
1. ユーザーはプロジェクト新規作成を選択する
2. 「サンプル建築を読込」オプションを選択する
3. サンプル事務所ビル（30m×20m、柱スパン6m×7m）が表示される
4. 外壁の方位・ガラス面積比のメタデータが自動設定される

**前提条件**:
- `sample-architecture.json` が `packages/core/src/data/` に配置済み

**優先度**: Must Have

## エピック2: ゾーニングと負荷計算

### ストーリー 2.1: HVACゾーンの作成 🔵

**信頼性**: 🔵 *PRDセクション11.2・7.1より*

**私は** 空調設備設計者 **として**
**フロア平面上でHVACゾーンの境界を描画し、用途・条件を設定したい**
**そうすることで** 負荷計算の基盤となるゾーン定義ができる

**関連要件**: REQ-201, REQ-202, REQ-203, REQ-204, REQ-205

**詳細シナリオ**:
1. ユーザーはZoneフェーズで zone_draw ツールを選択する
2. フロア平面上でポリゴンの頂点をクリックしてゾーン境界を描画する
3. 描画中に面積がリアルタイム表示される
4. 描画確定後、右パネルで用途（office_general等）、天井高、在室密度を設定する
5. ゾーンが用途に応じたカラーで半透明表示される

**前提条件**:
- HVACモードが有効
- Zoneフェーズが選択されている
- フロア（Level）が選択されている

**優先度**: Must Have

---

### ストーリー 2.2: ペリメータ外皮条件の入力 🔵

**信頼性**: 🔵 *PRDセクション11.2 v2追加より*

**私は** 空調設備設計者 **として**
**ゾーンのペリメータ（外壁面）条件を方位別に入力したい**
**そうすることで** 外皮負荷を考慮した正確な冷暖房負荷計算ができる

**関連要件**: REQ-205, REQ-208, REQ-209

**詳細シナリオ**:
1. ユーザーはゾーンを選択し、perimeter_edit ツールを使用する
2. 建築参照の外壁面と交差する辺について、方位・壁面積・ガラス面積比が半自動入力される
3. 自動検出できない場合、ユーザーが方位（N/NE/E/SE/S/SW/W/NW）を選び、壁面積・ガラス面積比を手動入力する
4. 右パネルに perimeterSegments が一覧表示される

**前提条件**:
- HvacZone が作成済み
- 建築参照データが存在する（半自動入力の場合）

**制約事項**:
- glazingRatio は 0.0〜1.0 の範囲
- 方位は8方位（N/NE/E/SE/S/SW/W/NW）

**優先度**: Must Have

### ストーリー 2.3: 負荷計算の実行と確認 🔵

**信頼性**: 🔵 *PRDセクション11.4・15.1より*

**私は** 空調設備設計者 **として**
**ゾーンごとの冷暖房負荷と必要風量を算出して確認したい**
**そうすることで** 系統構成と機器選定の基礎データが得られる

**関連要件**: REQ-301, REQ-302, REQ-303, REQ-304, REQ-305, REQ-306

**詳細シナリオ**:
1. ユーザーは負荷計算を実行する（load_calcツール）
2. 各ゾーンの冷房負荷・暖房負荷・必要風量が算出される
3. ゾーンのカラーがグレーから用途別色にフェードインする
4. 右パネルに内部負荷・外皮負荷（方位別内訳）が表示される
5. 「Z-01: 冷房 25.2kW（内部 18.0 + 外皮 7.2）、風量 7,560 m³/h」のように確認できる

**前提条件**:
- 1つ以上の HvacZone が作成済み
- ゾーンに用途と面積が設定済み

**優先度**: Must Have

## エピック3: 系統構成と機器配置

### ストーリー 3.1: ゾーングルーピング 🔵

**信頼性**: 🔵 *PRDセクション11.3 v2追加・ヒアリングQ7より*

**私は** 空調設備設計者 **として**
**複数のゾーンを1つのAHU系統にグルーピングしたい**
**そうすることで** 実務と同じN:1（複数ゾーン対1AHU）構成で機器選定ができる

**関連要件**: REQ-401, REQ-402, REQ-403, REQ-404, REQ-405

**詳細シナリオ**:
1. ユーザーはEquipフェーズで zone_grouping ツールを選択する
2. ゾーン一覧からチェックボックスで対象ゾーンを選択する
3. 「系統に追加」を実行すると SystemNode が作成される
4. 合算負荷（冷房65kW）と合算風量（19,500 m³/h）が即時表示される
5. ゾーンの枠線が系統カラーに変化する

**前提条件**:
- 負荷計算が完了した HvacZone が2つ以上存在する

**制約事項**:
- 1ゾーンは1系統にのみ所属可能

**優先度**: Must Have

---

### ストーリー 3.2: 機器候補の提示と採用 🔵

**信頼性**: 🔵 *PRDセクション11.5・15.3より*

**私は** 空調設備設計者 **として**
**合算負荷に基づいてAHU候補を見て、1台を採用・配置したい**
**そうすることで** 適切な容量の機器を選定し、ダクト接続の起点を確保できる

**関連要件**: REQ-501, REQ-502, REQ-503, REQ-504, REQ-505

**詳細シナリオ**:
1. ユーザーは系統を選択し、機器選定パネルを開く
2. 合算負荷65kW、合算風量19,500m³/hに対し、AHU-L-20000（140kW、20,000m³/h）が推奨候補として表示される
3. 他の候補（AHU-L-30000等）も容量過不足順にリスト表示される
4. ユーザーはAHU-L-20000を採用する
5. ビューポート上で配置位置をクリックし、AHUが3D表示される
6. ポート（給気、還気、外気、冷水供給、冷水還水）が表示される

**前提条件**:
- SystemNode が作成済み
- `catalog-ahu.json` が配置済み

**優先度**: Must Have

---

### ストーリー 3.3: 制気口の配置 🔵

**信頼性**: 🔵 *PRDセクション11.6より*

**私は** 空調設備設計者 **として**
**各ゾーンに制気口を配置し、風量を配分したい**
**そうすることで** ダクト接続の末端が確保できる

**関連要件**: REQ-601, REQ-602, REQ-603, REQ-604

**詳細シナリオ**:
1. ユーザーはEquipフェーズで diffuser_place ツールを選択する
2. カタログから制気口サイズ（例: ANEMO-400）を選択する
3. ゾーン内の天井面上でクリックして配置する
4. 配置された制気口にゾーンの必要風量が均等配分される
5. 各制気口に風量ラベルが表示される

**前提条件**:
- HvacZone が作成済み（負荷計算済み）
- `catalog-diffuser.json` が配置済み

**優先度**: Must Have

## エピック4: ダクトルーティングと計算

### ストーリー 4.1: ダクトの手動ルーティング 🔵

**信頼性**: 🔵 *PRDセクション11.7・11.8より*

**私は** 空調設備設計者 **として**
**AHUから制気口までダクトルートを手動で引きたい**
**そうすることで** 分岐を含む実際のルーティングを設計できる

**関連要件**: REQ-701, REQ-702, REQ-703, REQ-704, REQ-705

**詳細シナリオ**:
1. ユーザーはRouteフェーズで duct_route ツールを選択する
2. AHUの給気ポートをクリックしてダクト起点を設定する
3. 折点をクリックしながら幹線ダクトを描画する
4. T分岐を打ち、各ゾーンへ枝線を引く
5. 制気口のポートに近づくとスナップし、接続が確定する
6. 制気口手前の区間を選択し、ductMaterial を flexible に変更する
7. ダクトの色がグレー→青に変化する

**前提条件**:
- AHU と制気口が配置済み
- Routeフェーズが選択されている

**優先度**: Must Have

---

### ストーリー 4.2: 風量自動配分と寸法選定 🔵

**信頼性**: 🔵 *PRDセクション11.8・11.9・15.4・15.5より*

**私は** 空調設備設計者 **として**
**ダクト接続後に風量が自動配分され、寸法が自動選定されるのを確認したい**
**そうすることで** 手計算なしでダクトサイズの当たりが付けられる

**関連要件**: REQ-801, REQ-802, REQ-803, REQ-901, REQ-902, REQ-903

**詳細シナリオ**:
1. ダクト接続が完了すると、システムが末端から上流に向かって風量を自動配分する
2. 全ダクト区間の風量ラベルが一斉更新される
3. 幹線（19,500 m³/h）から枝線（各ゾーン向け）まで風量が確定する
4. 等速法で寸法が自動選定される（幹線: W600×H400等）
5. ダクトの太さが断面に比例して変化し、寸法ラベルが追加される

**前提条件**:
- AHUから全制気口までダクト接続が完了している

**優先度**: Must Have

---

### ストーリー 4.3: ダクト圧損計算 🔵

**信頼性**: 🔵 *PRDセクション11.10・15.6より*

**私は** 空調設備設計者 **として**
**ダクト系統の圧損計算結果を確認したい**
**そうすることで** ファン静圧の当たりが付けられ、設計の妥当性を判断できる

**関連要件**: REQ-1001, REQ-1002, REQ-1003, REQ-1004

**詳細シナリオ**:
1. ユーザーはCalcフェーズで pressure_loss ツールを実行する
2. 各区間の直管圧損と継手損失が計算される
3. フレキダクト区間は鋼板の4倍の摩擦係数で計算される
4. 最遠経路がハイライト表示される
5. 右パネルに系統総圧損と requiredFanPressure が表示される

**前提条件**:
- 風量自動配分と寸法選定が完了している

**優先度**: Must Have

## エピック5: 配管ルーティング

### ストーリー 5.1: 冷温水配管のルーティング 🔵

**信頼性**: 🔵 *PRDセクション11.11・15.7・ヒアリングQ7より*

**私は** 空調設備設計者 **として**
**AHUの冷温水配管ルートを引き、口径と圧損を確認したい**
**そうすることで** AHU系統のワンパスが完結する

**関連要件**: REQ-1101, REQ-1102, REQ-1103, REQ-1104, REQ-1105

**詳細シナリオ**:
1. ユーザーはRouteフェーズで pipe_route ツールを選択する
2. AHUの冷水供給ポートからクリックして配管起点を設定する
3. 折点を打ちながら配管ルートを描画する
4. 冷水供給配管（水色）と冷水還水配管（紺色）を引く
5. コイル能力から口径が自動算出される（例: 65A）
6. 等価長さ法で概算圧損が表示される

**前提条件**:
- AHU が配置済み
- AHU の冷房能力が設定済み

**優先度**: Must Have

### ストーリー 6.3: ワンパス完走（フルデモ） 🔵

**信頼性**: 🔵 *PRDセクション21.2・21.3より*

**私は** 空調設備設計者 **として**
**建築参照読込からバリデーション・保存まで一筆書きで完走したい**
**そうすることで** ツールの価値（設計の流れが切れない）を実感できる

**関連要件**: 全要件

**詳細シナリオ**:
1. サンプル建築を読込（30m×20m事務所ビル）
2. 3ゾーンを作成（南側事務室120m²、北側事務室180m²、会議室40m²）
3. 外皮条件を入力し、負荷計算を実行（合計 約65kW）
4. 3ゾーンを1系統にグルーピング
5. AHU-L-20000を採用・配置
6. 制気口9個を配置
7. ダクト幹線+3分岐+フレキを接続
8. 冷温水配管2本を接続
9. 再計算・バリデーション実行
10. 保存

**前提条件**:
- 全機能が実装済み
- サンプルデータセットが配置済み

**制約事項**:
- 30分以内に完走できること（初見、操作説明あり）

**優先度**: Must Have
