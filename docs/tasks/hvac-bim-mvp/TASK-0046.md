## 実装詳細

### 1. 🔵 ワンパスフロー通し実行（dataflow.md ワンパス全体フロー準拠）

dataflow.mdに定義されたワンパス全体フローを最初から最後まで通しで実行する。

- **Step 1**: 建築モデル上にHVACゾーンを3つ作成（事務室、会議室、サーバー室）
- **Step 2**: 各ゾーンの負荷計算を実行、calcResultに結果が格納される
- **Step 3**: 系統グルーピング（2系統に分類）
- **Step 4**: 各系統にAHUを配置、自動機器選定が動作
- **Step 5**: 各ゾーンにディフューザー（制気口）を配置
- **Step 6**: AHU→ディフューザーのダクトルーティング
- **Step 7**: AHUへの冷温水配管接続
- **Step 8**: 風量自動配分→寸法計算→圧損計算の一連実行
- **Step 9**: バリデーション実行、警告0件を確認
- **Step 10**: 保存→リロード→復元の確認

### 2. 🔵 各プリセットからの復帰テスト

TASK-0042で作成した5段階プリセットから作業を再開できることを検証する。

- preset-00-empty → ゾーン作成から開始して最後まで到達できる
- preset-01-zones → 系統グルーピングから開始して最後まで到達できる
- preset-02-equip → ダクトルーティングから開始して最後まで到達できる
- preset-03-route → 計算実行から開始して最後まで到達できる
- preset-04-complete → 完成状態の確認、警告なし、保存復元が正常

### 3. 🔵 NFR検証: 30fps維持(NFR-001), 再計算5秒以内(NFR-002)

非機能要件のパフォーマンス検証。

- **NFR-001**: Chrome DevToolsのPerformanceタブで30fps維持を確認
  - 3ゾーン・2系統・10ダクト区間・5配管のシーンで計測
  - カメラ回転、ズーム時のフレームレート確認
  - ノード選択・ホバー時のフレームレート確認
- **NFR-002**: 再計算カスケードの実行時間を計測
  - ゾーン条件変更→全カスケード完了まで5秒以内
  - console.timeで計測、結果を記録

### 4. 🟡 手計算との妥当性検証

計算結果の妥当性を手計算と比較して検証する。

- **NFR-301**: 負荷計算結果が手計算値と±20%以内
  - 事務室100m²の冷房負荷: 手計算値との比較
  - 方位補正係数の影響確認
- **NFR-302**: 圧損計算結果が手計算値と±15%以内
  - 直線ダクト10mの圧損: ダルシー・ワイスバッハ式との比較
  - 局部抵抗の影響確認
- **NFR-303**: 機器選定結果が手動選定と一致
  - 負荷に対して適切な能力の機器が選定されること

### 5. 🔵 エッジケース総合テスト

定義済みエッジケースの網羅的テスト。

- **EDGE-001**: ゾーン面積0m²の入力 → 適切なバリデーションエラー
- **EDGE-002**: ダクト長さ0mの区間 → 作成不可またはエラー表示
- **EDGE-003**: 循環ダクト接続（A→B→A） → 検出・警告表示
- **EDGE-004**: 機器能力不足 → 警告表示、代替候補提示
- **EDGE-005**: 風量合計が系統能力を超過 → airflow_mismatch警告
- **EDGE-101**: 全ノード削除後のシーン復帰 → 空シーンが正常に動作
- **EDGE-102**: 同一位置に複数ディフューザー配置 → 適切な処理
- **EDGE-103**: Undo/Redo後の計算整合性 → dirtyマーク→再計算→結果一致
