### 2. 🔵 建築参照外壁面との交差検出 → perimeterSegments半自動入力 (REQ-208)

ゾーン境界ポリゴンと建築参照の外壁面データを交差検出し、外皮条件を半自動で入力する。

```typescript
function detectPerimeterSegments(
  zoneBoundary: { x: number; y: number }[],
  architectureWalls: WallMetadata[]
): PerimeterSegment[] {
  const segments: PerimeterSegment[] = []

  for (const wall of architectureWalls) {
    // ゾーン境界の各辺と外壁面の交差判定
    const intersectionLength = calculateIntersection(zoneBoundary, wall.vertices)
    if (intersectionLength > 0) {
      segments.push({
        orientation: wall.orientation,
        wallArea: intersectionLength * wallHeight, // 交差長 × 壁高さ
        glazingRatio: wall.glazingRatio,
      })
    }
  }

  return segments
}
```

- 2Dラインセグメント交差判定アルゴリズム
- 交差長さ × 壁高さで壁面積を算出
- 建築参照のglazingRatioをそのまま引き継ぎ
- ユーザーは半自動結果を確認・修正可能

### 3. 🔵 手動入力UI: 方位選択 + 壁面積 + ガラス面積比 (REQ-209)

建築参照が存在しない場合の手動入力フォーム。

```typescript
function ManualInputPanel({
  segments,
  onUpdate,
}: {
  segments: PerimeterSegment[]
  onUpdate: (segments: PerimeterSegment[]) => void
}) {
  const addSegment = () => {
    onUpdate([
      ...segments,
      { orientation: 'S', wallArea: 0, glazingRatio: 0.3 },
    ])
  }

  return (
    <div>
      {segments.map((seg, i) => (
        <div key={i}>
          <select value={seg.orientation}>
            {/* N, NE, E, SE, S, SW, W, NW */}
          </select>
          <input type="number" value={seg.wallArea} min={0} />
          <input type="number" value={seg.glazingRatio} min={0} max={1} step={0.1} />
          <button onClick={() => removeSegment(i)}>削除</button>
        </div>
      ))}
      <button onClick={addSegment}>外壁面を追加</button>
    </div>
  )
}
```

- 方位: ドロップダウン（8方位）
- 壁面積: 数値入力（m²）
- ガラス面積比: 数値入力（0.0~1.0、ステップ0.1）
- セグメントの追加・削除が可能

### 4. 🔵 glazingRatio 0.0~1.0 バリデーション (EDGE-002)

ガラス面積比の範囲バリデーション。

```typescript
const PerimeterSegmentSchema = z.object({
  orientation: z.enum(['N', 'NE', 'E', 'SE', 'S', 'SW', 'W', 'NW']),
  wallArea: z.number().nonnegative(),
  glazingRatio: z.number().min(0).max(1),
})

function validateGlazingRatio(value: number): boolean {
  return value >= 0 && value <= 1
}
```

- 入力時にリアルタイムバリデーション
- 範囲外の値は赤色ハイライト + エラーメッセージ表示
- 保存時にZodスキーマでもバリデーション

### 5. 🔵 updateNode(hvacZoneId, {perimeterSegments}) → markDirty

外皮条件の保存処理。

```typescript
const handleSave = useCallback(() => {
  // バリデーション
  const validated = segments.every(
    (seg) => PerimeterSegmentSchema.safeParse(seg).success
  )
  if (!validated) return

  updateNode(zoneId, { perimeterSegments: segments })
  // updateNode内部でmarkDirtyが自動実行
  // → LoadCalcSystemがdirty検出 → 再計算
}, [segments, zoneId, updateNode])
```

- updateNodeはuseSceneのアクション
- 内部で自動的にdirtyNodesにマーク
- LoadCalcSystem（TASK-0015）がdirtyを検出して負荷再計算

## 注意事項

- **ツール配置**: packages/editorに配置。Three.js API直接呼び出し禁止
- **2D交差判定**: 交差検出は2D（XY平面）で実施。3D交差は不要
- **精度**: 浮動小数点演算の誤差に注意。交差長さの比較ではイプシロン（例: 0.001m）を考慮
- **undo対応**: updateNode経由のため、useSceneのundo/redo（Zundo 50-step）が自動的に機能する
- **建築参照依存**: TASK-0011の建築参照ローダーに依存。建築参照が未読込の場合は手動入力モードにフォールバック
