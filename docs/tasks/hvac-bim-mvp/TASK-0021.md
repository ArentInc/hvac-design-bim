## タスク概要

系統の合算負荷からAHUカタログを検索し、適合する機器候補を提示するシステム。フィルタ条件は「定格容量 >= 合算負荷 x 余裕率」かつ「定格風量 >= 合算風量 x 余裕率」。余裕率のデフォルトは1.1でユーザー変更可能。候補は容量過不足の少ない順にソートされる。

### 2. 🔵 容量・風量フィルタ (REQ-502)

カタログからcapacityとairflowの条件を満たす機器をフィルタする。

```typescript
interface AhuCatalogEntry {
  id: string
  name: string
  coolingCapacity: number  // kW
  heatingCapacity: number  // kW
  ratedAirflow: number     // m3/h
  // ...その他スペック
}

const filterCandidates = (
  catalog: AhuCatalogEntry[],
  totalCoolingLoad: number,
  totalAirflow: number,
  margin: number
): AhuCatalogEntry[] => {
  return catalog.filter(entry =>
    entry.coolingCapacity >= totalCoolingLoad * margin &&
    entry.ratedAirflow >= totalAirflow * margin
  )
}
```

- 冷房容量と風量の両方を満たす機器のみ候補に残す
- margin（余裕率）を乗じた値で比較

### 3. 🔵 容量過不足順ソート

候補を容量過不足の少ない順（最適な機器が上位）にソートする。

```typescript
const sortCandidates = (
  candidates: AhuCatalogEntry[],
  totalCoolingLoad: number,
  margin: number
): AhuCatalogEntry[] => {
  return candidates.sort((a, b) => {
    const overA = a.coolingCapacity - totalCoolingLoad * margin
    const overB = b.coolingCapacity - totalCoolingLoad * margin
    return overA - overB  // 過不足が小さい順
  })
}
```

- 必要容量に対する余剰が小さい機器を優先
- 同一余剰の場合は風量余剰で二次ソート

### 4. 🔵 余裕率のデフォルトとユーザー変更 (REQ-503)

余裕率はデフォルト1.1（10%余裕）で、ユーザーが変更可能にする。

```typescript
const DEFAULT_MARGIN = 1.1

// SystemNodeに余裕率を保持
interface SystemNode {
  // ...
  selectionMargin?: number  // デフォルト1.1
}

const margin = systemNode.selectionMargin ?? DEFAULT_MARGIN
```

- SystemNodeのselectionMarginフィールドで系統ごとに余裕率を管理
- 未設定時はDEFAULT_MARGIN（1.1）を使用
- UIからの変更はupdateNode(systemId, {selectionMargin: value})で反映

### 5. 🟡 候補なし時の処理 (EDGE-005)

フィルタ結果が0件の場合の処理を実装する。

```typescript
const selectEquipment = (systemId: string) => {
  const candidates = filterAndSortCandidates(...)

  if (candidates.length === 0) {
    updateNode(systemId, {
      equipmentCandidates: [],
      selectionStatus: 'no-candidates',
    })
    // UIで「候補なし — 手動設定を許可」を表示
    return
  }

  updateNode(systemId, {
    equipmentCandidates: candidates.map(c => c.id),
    selectionStatus: 'candidates-available',
    recommendedEquipmentId: candidates[0].id,
  })
}
```

- selectionStatus: 'no-candidates' で候補なし状態を明示
- 候補なし時もエラーにはせず、手動設定を許可するフラグを設定
- UIパネル（TASK-0027）で「候補なし」表示と手動入力フォームを提供

## 単体テスト要件

### テスト1: 基本フィルタ — 条件を満たす候補
- **信頼性**: 🔵
- **Given**: カタログに3機種（容量50kW/80kW/120kW）、合算負荷70kW、余裕率1.1
- **When**: フィルタを実行
- **Then**: 80kWと120kWの2機種が候補に残る（50kW < 70 x 1.1 = 77kW）

### テスト2: 風量フィルタ
- **信頼性**: 🔵
- **Given**: 容量は全機種OK、風量が10000/15000/20000 m3/h、必要風量12000 m3/h、余裕率1.1
- **When**: フィルタを実行
- **Then**: 15000と20000の2機種が候補に残る（10000 < 12000 x 1.1 = 13200）

### テスト3: 容量過不足順ソート
- **信頼性**: 🔵
- **Given**: 候補が80kW, 120kW, 100kW、合算負荷70kW、余裕率1.1
- **When**: ソートを実行
- **Then**: 80kW → 100kW → 120kWの順（余剰が小さい順）

### テスト4: デフォルト余裕率1.1
