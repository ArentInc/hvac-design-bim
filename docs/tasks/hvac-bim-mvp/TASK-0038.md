## 実装詳細

### 1. 🔵 ValidationSystemコンポーネント作成

**ファイル**: `packages/core/src/systems/hvac/validation-system.tsx`

useFrameループ内でdirtyノードを検出し、全HVACノードを走査して警告配列を生成するReactコンポーネント（renderはnull）。

- Coreシステムパターンに従い、renderingやThree.js依存なし
- dirtyNodes検出→全ノード走査→warnings[]生成→ストア更新
- 警告配列はuseSceneのcalcResult内またはvalidation専用フィールドに格納

### 2. 🔵 Warning型定義

**ファイル**: `packages/core/src/schema/hvac/warning.ts`

```typescript
type WarningCode =
  | 'unconnected_port'
  | 'airflow_not_set'
  | 'size_not_determined'
  | 'velocity_exceeded'
  | 'pressure_not_calculated'
  | 'zone_no_system'
  | 'airflow_mismatch'
  | 'pipe_not_connected'

type WarningSeverity = 'error' | 'warning' | 'info'

interface Warning {
  id: string
  nodeId: string
  nodeType: string
  severity: WarningSeverity
  code: WarningCode
  message: string
}
```

### 3. 🔵 8警告チェック実装 (REQ-1201)

各チェックを独立した関数として実装する。

- **unconnected_port**: 機器（AHU等）のポートに接続先ノードが設定されていないものを検出
- **airflow_not_set**: ダクト区間またはディフューザーのairflow_m3hが0または未設定
- **size_not_determined**: ダクト区間のwidth/heightまたは配管のdiameterが0
- **velocity_exceeded**: ダクト区間の風速が推奨値を超過（主ダクト15m/s、枝ダクト10m/s等）
- **pressure_not_calculated**: ダクトルートまたは配管ルートの圧損計算が未実施（calcResult未生成）
- **zone_no_system**: ゾーンがいずれの系統にも割り当てられていない
- **airflow_mismatch**: 系統に接続された制気口の風量合計と系統の設計風量の乖離が±5%超
- **pipe_not_connected**: 配管ノードの接続先が未設定

### 4. 🔵 dirty検出→全ノード走査→warnings[]生成

- useFrame内でdirtyNodesにHVACノードが含まれるか確認
- dirty検出時に全HVACノードを走査し、8種のチェック関数を適用
- 生成されたWarning配列をストアに格納
- 前回の警告と差分がない場合は更新をスキップ（パフォーマンス対策）

### テスト2: airflow_not_set検出
- **信頼性**: 🔵
- **Given**: DuctSegmentNodeのairflow_m3hが0
- **When**: ValidationSystemがノードを走査
- **Then**: code='airflow_not_set' のWarningが生成される

### テスト3: size_not_determined検出
- **信頼性**: 🔵
- **Given**: DuctSegmentNodeのwidth=0, height=0
- **When**: ValidationSystemがノードを走査
- **Then**: code='size_not_determined' のWarningが生成される

### テスト4: velocity_exceeded検出
- **信頼性**: 🔵
- **Given**: ダクト区間の計算済み風速が推奨値を超過
- **When**: ValidationSystemがノードを走査

### テスト5: zone_no_system検出

## 注意事項

- **Coreパッケージ**: Three.jsやviewer依存は禁止。純粋なデータ検証ロジックのみ
- **パフォーマンス**: ノード数が増えた場合のフレーム毎走査コストに注意。dirty検出時のみ走査を実行
- **閾値管理**: airflow_mismatch の±5%等の閾値は定数として管理し、将来の設定変更に対応可能にする
- **severity設定**: unconnected_port, pipe_not_connected は 'error'、velocity_exceeded は 'warning'、pressure_not_calculated は 'info' 等、適切な重要度を設定
