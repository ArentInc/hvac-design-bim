## タスク概要

TASK-0002~0004で定義した7つのHVACノード（hvac_zone, system, ahu, diffuser, duct_segment, duct_fitting, pipe_segment）をAnyNode判別共用体に追加し、LevelNodeのchildren型を拡張し、sceneRegistryのbyTypeマップに7タイプを追加する。これにより既存のノード管理システム（createNode/updateNode/deleteNode、dirty node system）がHVACノードに対してもシームレスに動作するようになる。

## 依存タスク

## 実装詳細

### 1. 🔵 AnyNode union に7HVACノードを追加

**ファイル**: `packages/core/src/schema/types.ts`

既存の AnyNode 判別共用体（discriminated union on `type` field）に以下を追加:

```typescript
// 既存: WallNode | SlabNode | CeilingNode | RoofNode | ZoneNode | ...
// 追加:
| HvacZoneNode
| SystemNode
| AhuNode
| DiffuserNode
| DuctSegmentNode
| DuctFittingNode
| PipeSegmentNode
```

- 判別フィールド: `type`（既存パターン準拠）
- 各ノードのimport追加

### 2. 🔵 LevelNode スキーマの children 拡張

**ファイル**: `packages/core/src/schema/nodes/level.ts`（既存）

LevelNode の children 配列が受け入れるIDタイプを拡張。既存の階層構造（CLAUDE.md参照: Level → {Wall, Slab, Ceiling, Roof, Zone, Scan, Guide}）に7HVACタイプを追加:

- Level → {Wall, Slab, Ceiling, Roof, Zone, Scan, Guide, **HvacZone, System, Ahu, Diffuser, DuctSegment, DuctFitting, PipeSegment**}

注意: children はstring[]（ID配列）なので型制約は緩いが、createNode時のparentId検証ロジックでLevel以外への追加を防止する必要がある。

### 3. 🔵 sceneRegistry.byType に7タイプ追加

**ファイル**: `packages/viewer/src/utils/scene-registry.ts`（または該当ファイル）

byType マップに新しいエントリを追加:

```typescript
byType: {
  // 既存...
  wall: Set<string>,
  slab: Set<string>,
  // ...
  // 追加:
  hvac_zone: Set<string>,
  system: Set<string>,
  ahu: Set<string>,
  diffuser: Set<string>,
  duct_segment: Set<string>,
  duct_fitting: Set<string>,
  pipe_segment: Set<string>,
}
```

### 4. 🔵 ノードタイプ定数の追加

**ファイル**: `packages/core/src/schema/types.ts` または `packages/core/src/constants.ts`

```typescript
export const HVAC_NODE_TYPES = [
  'hvac_zone', 'system', 'ahu', 'diffuser',
  'duct_segment', 'duct_fitting', 'pipe_segment',
] as const

export const ALL_NODE_TYPES = [...EXISTING_NODE_TYPES, ...HVAC_NODE_TYPES] as const
```

## 注意事項

- **後方互換性が最重要**: 既存14ノードタイプの動作に影響を与えないこと。既存の全テスト（建築ノード関連）がpassし続けることを確認
- **判別共用体のパフォーマンス**: Zod 4のdiscriminated unionは `type` フィールドで高速判別するため、ノード数増加のパフォーマンス影響は軽微
- **sceneRegistry初期化**: byType の新エントリは空の Set<string> で初期化。レンダラー実装前でもエラーにならないこと
- **IndexedDB永続化**: useScene は IndexedDB に永続化されるため、新ノードタイプのシリアライズ/デシリアライズが正しく動作することを確認
- **createNode のparentId検証**: HVACノードの親はLevelNodeに限定。Building直下やSite直下への追加はエラーにする
