## 実装詳細

### 1. 🔵 WarningListPanelコンポーネント作成

**ファイル**: `packages/editor/src/components/ui/sidebar/panels/warning-list-panel.tsx`

editorパッケージ内に警告一覧UIコンポーネントを作成する。

- props: なし（ストアから直接警告配列を取得）
- ValidationSystemが生成したwarnings配列をサブスクライブ
- editorパッケージ内のため、useEditor/useViewerの参照が可能

### 2. 🔵 全警告一覧（severity別ソート、ノード名、警告メッセージ）

警告配列をseverityの重要度順にソートして表示する。

- ソート順: error > warning > info
- 各行の表示内容:
  - severityアイコン（error: 赤丸、warning: 黄三角、info: 青i）
  - ノード名（nodes[warning.nodeId].nameまたはnodeType+短縮ID）
  - 警告メッセージ（warning.message）
- 警告0件時は「警告なし」のプレースホルダー表示

### 3. 🔵 クリック→ useViewer.selectedIds更新 + カメラズーム (REQ-1203)

警告行クリック時の選択・ズーム動作。

- クリック時にuseViewer.getState().setSelectedIds([warning.nodeId])を呼び出し
- sceneRegistry.nodes.get(warning.nodeId)でObject3Dを取得
- Object3DのBoundingBoxを計算し、カメラをフィットズーム
- 選択パスの親（building/level/zone）も適切に設定

### 4. 🔵 右パネル詳細表示連動 (REQ-1202)

ノード選択により右パネルのプロパティパネルが自動的に該当ノードの詳細を表示する。

- selectedIds更新により既存のPropertyPanelが自動連動
- 追加の警告詳細情報（警告コード、推奨対策）を表示
- 複数警告がある場合はノードの全警告をリスト表示

### 5. 🟡 警告種別別フィルタリング

警告コード別のフィルタリング機能。

- WarningCode別のトグルフィルタ
- フィルタ状態はローカルstate管理
- フィルタ適用後もソート順を維持

## 注意事項

- **editorパッケージ**: このコンポーネントはeditorパッケージに配置。viewerパッケージへの配置は不可
- **選択パス**: ノード選択時にbuildingId/levelId/zoneIdの親パスも正しく設定する必要がある
- **カメラズーム**: viewerが公開するカメラ制御APIを使用する。直接Three.jsカメラを操作しない
- **大量警告**: 警告数が多い場合のスクロール性能に注意。仮想リスト化は将来対応
