## 実装詳細

### 1. 🔵 WarningBadgeSystemコンポーネント作成

**ファイル**: `packages/viewer/src/systems/hvac/warning-badge-system.tsx`

Viewerシステムパターンに従い、useFrame内でwarnings配列を参照し、sceneRegistryのObject3Dにバッジを付与するコンポーネント。

- renderはnull（Systemsパターン）
- useFrame内で警告状態の変化を検出しバッジを更新
- viewerパッケージ内のため、Three.jsのSprite/CanvasTexture使用可

### 2. 🔵 sceneRegistry経由でノードのObject3D取得

sceneRegistry.nodesからnodeIdに対応するObject3Dを取得する。

- `sceneRegistry.nodes.get(nodeId)` でObject3D参照を取得
- Object3Dが存在しないノードはスキップ
- ノードの削除時にバッジも自動的にクリーンアップ

### 3. 🔵 ノードに紐づくwarning数をカウント

ValidationSystem（TASK-0038）が生成したwarnings配列からnodeId別にカウントする。

- warnings配列をnodeIdでグルーピング
- ノードごとの警告件数を算出
- severity別にカウントを分離（将来のバッジ色分け対応用）

### 4. 🔵 赤丸+数字のSprite/Billboard表示 (REQ-1202)

CanvasTextureで赤丸に白数字のバッジを生成し、Spriteとしてノードの上部に配置する。

- Canvas2Dで赤丸（#E53935）を描画し白文字で警告数を表示
- SpriteMaterial + Sprite でBillboard表示（常にカメラ正面を向く）
- ノードのBoundingBox上端から少し上にオフセット配置
- バッジサイズはカメラ距離に依存しない固定スケール

### 5. 🟡 警告0件時はバッジ非表示

警告が解消された場合のバッジ除去処理。

- ノードの警告が0件になったらSpriteをObject3Dから除去
- 次フレームで再度警告が付与された場合に再表示
- メモリリーク防止のためTexture/Materialの適切なdispose

- **Then**: nodeAのObject3DにSpriteが子として追加され、数字「2」が表示される

### テスト2: 警告なしノードにバッジ非表示
- **信頼性**: 🔵
- **Given**: nodeBに警告がない
- **When**: WarningBadgeSystemがuseFrameで更新
- **Then**: nodeBのObject3DにSpriteが追加されていない

### テスト3: 警告解消時のバッジ除去
- **信頼性**: 🟡
- **Given**: nodeAに警告があり、バッジが表示されている
- **When**: nodeAの警告が0件に変化し、次のuseFrameで更新
- **Then**: nodeAのObject3DからSpriteが除去される

### テスト4: 複数ノードへの同時バッジ表示
- **信頼性**: 🔵
- **Given**: nodeA（3件）、nodeB（1件）に警告がある
- **When**: WarningBadgeSystemがuseFrameで更新
- **Then**: 両ノードにそれぞれ正しい数字のバッジが表示される

### テスト5: sceneRegistryに未登録ノードのスキップ
- **信頼性**: 🔵
- **Given**: 警告があるがsceneRegistryにObject3Dが登録されていないノード
- **When**: WarningBadgeSystemがuseFrameで更新
