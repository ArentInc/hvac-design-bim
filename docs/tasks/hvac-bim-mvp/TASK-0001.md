## 実装詳細

### 1. 🟡 vitest + 関連パッケージのインストール

vitestおよびテストに必要な関連パッケージをモノレポルートにインストールする。

```bash
bun add -d vitest @testing-library/react happy-dom
```

- `vitest`: テストランナー本体
- `@testing-library/react`: Reactコンポーネントテスト用（packages/editor, packages/viewerで必要）
- `happy-dom`: jsdomの軽量代替。ブラウザ環境シミュレーション

### 2. 🟡 各パッケージにvitest.config.ts作成

各パッケージのルートに `vitest.config.ts` を作成する。

**packages/core/vitest.config.ts**:
- environment: 'node'（UIなし、純粋ロジック）
- include: `['src/**/*.test.ts']`
- tsconfig参照: 各パッケージのtsconfig.json

**packages/viewer/vitest.config.ts**:
- environment: 'happy-dom'（Three.js関連のDOM依存）
- include: `['src/**/*.test.ts', 'src/**/*.test.tsx']`
- Three.jsモック設定が必要な場合のsetupFiles

**packages/editor/vitest.config.ts**:
- environment: 'happy-dom'（React UI コンポーネント）
- include: `['src/**/*.test.ts', 'src/**/*.test.tsx']`
- setupFiles: testing-library設定

### 3. 🟡 turbo.jsonにtestタスク追加

```json
{
  "tasks": {
    "test": {
      "dependsOn": ["^build"],
      "inputs": ["src/**/*.ts", "src/**/*.tsx", "vitest.config.ts"],
      "outputs": [],
      "cache": false
    }
  }
}
```

- `dependsOn: ["^build"]`: テスト前にビルド依存パッケージを構築
- `cache: false`: テストはキャッシュしない（状態依存の可能性）

### 4. 🟡 package.jsonにtestスクリプト追加

**ルートpackage.json**:
```json
{
  "scripts": {
    "test": "turbo test",
    "test:core": "turbo test --filter=@pascal-app/core",
    "test:viewer": "turbo test --filter=@pascal-app/viewer",
    "test:editor": "turbo test --filter=@pascal-app/editor"
  }
}
```

**各パッケージのpackage.json**:
```json
{
  "scripts": {
    "test": "vitest run",
    "test:watch": "vitest"
  }
}
```

### 5. 🟡 packages/coreで基本的なサンプルテスト作成・動作確認

`packages/core/src/__tests__/sample.test.ts` を作成し、基本動作を確認する。

```typescript
import { describe, expect, it } from 'vitest'

describe('vitest setup verification', () => {
  it('基本的なアサーションが動作する', () => {
    expect(1 + 1).toBe(2)
  })

  it('Zodスキーマのインポートが動作する', () => {
    // 既存スキーマの簡単な検証
    // 例: WallNodeのインポートと基本parse
  })
})
```

## 注意事項

- **Bunとの互換性**: Bunのテストランナー(`bun test`)ではなくvitestを使用する。vitestの方がエコシステムが充実しており、@testing-library等との統合が容易
- **Three.jsモック**: packages/viewerのテストではWebGL関連のモックが必要になる可能性がある。本タスクでは基本設定のみ行い、具体的なモック戦略はレンダラーテスト実装時に対応
- **パフォーマンス**: turboのキャッシュはテストタスクでは無効にする。テスト結果は入力ファイル以外の要因（環境変数等）に依存する可能性があるため
- **既存Biome設定との整合**: テストファイルがBiomeのlint/formatルールに準拠するよう確認
