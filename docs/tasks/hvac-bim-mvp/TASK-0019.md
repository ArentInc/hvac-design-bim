## タスク概要

SystemNodeのservedZoneIdsから各ゾーンのloadResultを合算し、系統全体の負荷を算出するシステム。dirtyノード検出により非同期で再計算を行い、updateNodeで即時更新する。ゾーン負荷が変更された場合も自動的に再集計がトリガーされる。

### 2. 🔵 冷房負荷合算 (REQ-405)

totalCoolingLoad = Σ zone.calcResult.coolingLoad を計算する。

```typescript
const totalCoolingLoad = servedZoneIds.reduce((sum, zoneId) => {
  const zone = nodes[zoneId]
  if (zone?.type === 'hvacZone' && zone.calcResult) {
    return sum + zone.calcResult.coolingLoad
  }
  return sum
}, 0)
```

- servedZoneIdsに含まれる各ゾーンのcalcResult.coolingLoadを合算
- ゾーンが存在しない場合やcalcResultが未計算の場合は0として扱う

### 3. 🔵 暖房負荷合算

### 4. 🔵 必要風量合算

totalAirflow = Σ zone.calcResult.requiredAirflow を計算する。

```typescript
const totalAirflow = servedZoneIds.reduce((sum, zoneId) => {
```

- 全ゾーンのrequiredAirflowを合算し、系統全体の必要風量を算出

### 5. 🔵 即時更新 (REQ-404)

updateNode(systemId, {aggregatedLoad}) で系統ノードを即時更新する。

```typescript
updateNode(systemId, {
  aggregatedLoad: {
    totalCoolingLoad,
    totalHeatingLoad,
    totalAirflow,
  },
})
```

- 計算結果はSystemNodeのaggregatedLoadフィールドに格納
- updateNodeによりdirtyフラグが自動的にセットされ、後続システムへ伝播

### 6. 🔵 ゾーン負荷変更時の再集計トリガー (REQ-1802)

ゾーンの負荷が変更された場合、そのゾーンを含むSystemNodeを検出し再集計する。

```typescript

for (const dirtyId of dirtyNodes) {
  const dirtyNode = nodes[dirtyId]
  if (dirtyNode?.type === 'hvacZone') {
    // このゾーンを含むsystemノードを検索
    for (const node of Object.values(nodes)) {
      if (node.type === 'system' && node.servedZoneIds?.includes(dirtyId)) {
        // このsystemノードを再計算対象に追加
      }
    }
  }
}
```

- hvacZoneがdirtyになった場合、そのゾーンを参照するSystemNodeを自動検出
- 検出されたSystemNodeに対して負荷再集計を実行

## 注意事項

- **coreパッケージに配置**: 純粋な計算ロジックのため、viewerやeditorからインポートしない
- **useFrameの実行頻度**: 毎フレーム実行されるため、dirtyノードがない場合は早期リターンでパフォーマンスを確保
- **浮動小数点精度**: 負荷合算時の浮動小数点誤差に注意。テストでは適切な許容範囲を設定
- **循環参照防止**: ゾーン→システム→ゾーンの参照で循環的なdirty伝播が起きないよう注意
