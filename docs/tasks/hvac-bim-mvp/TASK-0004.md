## タスク概要

ダクト2種（DuctSegmentNode, DuctFittingNode）と配管（PipeSegmentNode）の Zod スキーマを定義する。ダクトセグメントは直線区間、フィッティングはエルボ・分岐等の継手、パイプセグメントは冷温水配管の直線区間を表す。これらはルーティングツール（Phase 3）で自動生成され、圧損計算の対象となる。

## 実装詳細

### 1. 🔵 DuctSegmentNode Zod スキーマ定義

**ファイル**: `packages/core/src/schema/nodes/duct-segment.ts`

interfaces.ts の DuctSegmentNode 型定義と REQ-702~703 に基づく。

- **id prefix**: `'duct_seg'`
- **type**: `'duct_segment'`
- **フィールド**:
  - `start`: `[number, number, number]` — 始点座標
  - `end`: `[number, number, number]` — 終点座標
  - `medium`: DuctMedium enum — ダクト媒体
  - `shape`: DuctShape enum — 断面形状
  - `width`: number | null — 矩形ダクト幅 (mm)、shape='round' 時 null
  - `height`: number | null — 矩形ダクト高さ (mm)、shape='round' 時 null
  - `diameter`: number | null — 丸ダクト径 (mm)、shape='rectangular' 時 null
  - `ductMaterial`: DuctMaterial enum — ダクト材質
  - `airflowRate`: number | null — 風量 (m³/h)、自動配分前 null
  - `startPortId`: string — 始点ポートID
  - `endPortId`: string — 終点ポートID
  - `systemId`: string — 所属系統ID
  - `calcResult`: DuctCalcResult | null — 圧損計算結果

- **DuctMedium enum**: `'supply_air' | 'return_air' | 'exhaust_air'`
- **DuctShape enum**: `'rectangular' | 'round'`
- **DuctMaterial enum**: `'galvanized_steel' | 'stainless_steel' | 'aluminum' | 'flexible'`

- **DuctCalcResult サブスキーマ**:
  - `velocity`: number — 風速 (m/s)
  - `frictionLoss`: number — 摩擦損失 (Pa/m)
  - `totalPressureLoss`: number — 総圧損 (Pa)

## 注意事項

- **矩形/丸の排他性**: DuctSegmentのwidth/heightとdiameterはshapeに応じて排他的だが、Zodスキーマレベルではrefineで制約するか、shape別のdiscriminated unionにするか設計判断が必要。シンプルにnullableとし、ビジネスロジックで検証を推奨
- **アスペクト比**: 矩形ダクトのアスペクト比（width:height）は4:1以下が推奨。ヘルパー関数としてoptionalに提供可能だが、スキーマ制約には含めない
- **Port型再利用**: DuctFittingNodeのportsはTASK-0003で定義したPort型を使用
- **CalcResult初期値**: DuctCalcResult, PipeCalcResult は初期null。圧損計算システム（Phase 4）が値を設定
- **座標単位**: position/start/end はメートル単位、ダクトサイズ（width/height/diameter）はミリメートル単位
