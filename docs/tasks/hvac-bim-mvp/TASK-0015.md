## 実装詳細

### 1. 🔵 packages/core/src/systems/hvac/load-calc-system.tsx

Coreシステムとして実装。レンダリングはnull、ロジックはuseFrame内で実行。

```typescript
import { useScene } from '@pascal-app/core'

export function LoadCalcSystem() {
  const dirtyNodes = useScene((s) => s.dirtyNodes)
  const nodes = useScene((s) => s.nodes)
  const updateNode = useScene((s) => s.updateNode)

  useFrame(() => {
    // dirtyNodesからhvac_zoneタイプを抽出
    const dirtyZones = [...dirtyNodes].filter(
      (id) => nodes[id]?.type === 'hvac_zone'
    )

    if (dirtyZones.length === 0) return

    // requestIdleCallbackで非同期実行
    requestIdleCallback(() => {
      for (const zoneId of dirtyZones) {
        const zone = nodes[zoneId] as HvacZoneNode
        const result = calculateZoneLoad(zone)
        updateNode(zoneId, { calcResult: result })
      }
    })
  })

  return null
}
```

- Systemsはnullを返す（CLAUDE.md準拠）
- useFrameでdirtyNodes監視
- バッチ処理で複数ゾーンを一括計算

### 2. 🔵 internalLoad = floorArea × coolingLoadPerArea[usage] (REQ-302)

用途別冷房負荷原単位による内部負荷算出。

```typescript
// 冷房負荷原単位 (W/m²)
const COOLING_LOAD_PER_AREA: Record<string, number> = {
  office_general: 150,    // 一般オフィス
  conference: 200,        // 会議室
  reception: 120,         // 受付/ロビー
  office_server: 500,     // サーバー室
  corridor: 80,           // 廊下
}

function calculateInternalLoad(
  floorArea: number,
  usage: string
): number {
  const loadPerArea = COOLING_LOAD_PER_AREA[usage] ?? 150 // デフォルト値
  return floorArea * loadPerArea // W
}
```

- 用途不明時はoffice_generalのデフォルト値（150 W/m²）を適用
- 結果はW（ワット）単位

### 3. 🔵 envelopeLoad = Σ(wallArea × glazingRatio × solarCoeff × baseEnvelopeLoad) (REQ-303)

方位別日射補正係数による外皮負荷算出。

```typescript
// 方位別日射補正係数
const SOLAR_COEFFICIENT: Record<string, number> = {
  S: 1.0,
  SE: 1.1,
  SW: 1.1,
  E: 1.2,
  W: 1.2,
  NE: 0.8,
  NW: 0.8,
  N: 0.6,
}

const BASE_ENVELOPE_LOAD = 200 // W/m² — ベース外皮負荷

function calculateEnvelopeLoad(
  perimeterSegments: PerimeterSegment[]
): { total: number; breakdown: Record<string, number> } {
  const breakdown: Record<string, number> = {}
  let total = 0

  for (const seg of perimeterSegments) {
    const coeff = SOLAR_COEFFICIENT[seg.orientation] ?? 1.0
    const load = seg.wallArea * seg.glazingRatio * coeff * BASE_ENVELOPE_LOAD
    breakdown[seg.orientation] = (breakdown[seg.orientation] ?? 0) + load
    total += load
  }

  return { total, breakdown }
}
```

- 各perimeterSegmentの壁面積 × ガラス面積比 × 日射補正係数 × ベース外皮負荷
- 方位別内訳（breakdown）も同時算出
- perimeterSegmentsが空の場合、envelopeLoad = 0

### 4. 🔵 coolingLoad = internalLoad + envelopeLoad

冷房負荷の合計算出。

```typescript
function calculateCoolingLoad(
  internalLoad: number,
  envelopeLoad: number
): number {
  return internalLoad + envelopeLoad // W
}
```

### 5. 🔵 requiredAirflow = coolingLoad / (1.2 × 1005 × supplyAirTempDiff) → m³/h変換 (REQ-304)

必要風量の算出。

```typescript
const AIR_DENSITY = 1.2 // kg/m³
const AIR_SPECIFIC_HEAT = 1005 // J/(kg·K)

function calculateRequiredAirflow(
  coolingLoad: number,
  supplyAirTempDiff: number = 10 // K — デフォルト送風温度差
): number {
  if (supplyAirTempDiff <= 0) return 0

  const airflowM3s = coolingLoad / (AIR_DENSITY * AIR_SPECIFIC_HEAT * supplyAirTempDiff)
  const airflowM3h = airflowM3s * 3600 // m³/s → m³/h
  return Math.round(airflowM3h) // 整数に丸め
}
```

- 送風温度差デフォルト: 10K（設計条件から取得可能）
- 結果はm³/h単位
- 温度差0以下の場合は0を返す（ゼロ除算防止）

### 6. 🔵 perimeterLoadBreakdown: 方位別内訳 (REQ-306)

外皮負荷の方位別内訳を出力する。

```typescript
interface PerimeterLoadBreakdown {
  orientation: string
  load: number // W
  percentage: number // %
}

function formatBreakdown(
  breakdown: Record<string, number>,
  totalEnvelopeLoad: number
): PerimeterLoadBreakdown[] {
  return Object.entries(breakdown).map(([orientation, load]) => ({
    orientation,
    load,
    percentage: totalEnvelopeLoad > 0 ? (load / totalEnvelopeLoad) * 100 : 0,
  }))
}
```

- 各方位の負荷値（W）と全体に対する割合（%）
- TASK-0016のCalcResultPanelで表示

### 7. 🔵 heatingLoad計算（暖房原単位ベース）

暖房負荷の算出。

```typescript
// 暖房負荷原単位 (W/m²)
const HEATING_LOAD_PER_AREA: Record<string, number> = {
  office_general: 100,
  conference: 120,
  reception: 90,
  office_server: 0,       // サーバー室は暖房不要
  corridor: 60,
}

function calculateHeatingLoad(
  floorArea: number,
  usage: string
): number {
  const loadPerArea = HEATING_LOAD_PER_AREA[usage] ?? 100
  return floorArea * loadPerArea // W
}
```

- サーバー室（office_server）は暖房負荷0（発熱量で十分）
- 外皮からの熱損失は将来的に追加可能

### 8. 🔵 非同期実行: requestIdleCallback で UI非ブロッキング

大量のゾーン計算時にメインスレッドをブロックしない。

```typescript
function scheduleCalculation(callback: () => void) {
  if (typeof requestIdleCallback !== 'undefined') {
    requestIdleCallback(callback, { timeout: 100 })
  } else {
    // フォールバック（SSR等）
    setTimeout(callback, 0)
  }
}
```

- requestIdleCallback利用（ブラウザのアイドル時間に実行）
- timeout: 100msで最低実行保証
- SSR環境ではsetTimeoutにフォールバック

### 9. 🔵 dirty検出 → 計算 → updateNode(calcResult) → clearDirty

dirty nodeシステムとの統合。

```typescript
// 計算結果の型
interface CalcResult {
  coolingLoad: number       // W
  heatingLoad: number       // W
  requiredAirflow: number   // m³/h
  internalLoad: number      // W
  envelopeLoad: number      // W
  perimeterLoadBreakdown: PerimeterLoadBreakdown[]
  calculatedAt: number      // timestamp
}

function calculateZoneLoad(zone: HvacZoneNode): CalcResult {
  const internalLoad = calculateInternalLoad(zone.floorArea, zone.usage)
  const { total: envelopeLoad, breakdown } = calculateEnvelopeLoad(
    zone.perimeterSegments ?? []
  )
  const coolingLoad = calculateCoolingLoad(internalLoad, envelopeLoad)
  const requiredAirflow = calculateRequiredAirflow(
    coolingLoad,
    zone.designConditions?.supplyAirTempDiff
  )
  const heatingLoad = calculateHeatingLoad(zone.floorArea, zone.usage)

  return {
    coolingLoad,
    heatingLoad,
    requiredAirflow,
    internalLoad,
    envelopeLoad,
    perimeterLoadBreakdown: formatBreakdown(breakdown, envelopeLoad),
    calculatedAt: Date.now(),
  }
}
```

- 計算完了後、updateNode(zoneId, {calcResult})で結果保存
- updateNode内部でclearDirtyが自動実行
- calculatedAtで最終計算時刻を記録

## 注意事項

- **Coreシステムルール**: packages/coreに配置。Three.jsインポート禁止、viewerへの依存禁止
- **計算精度**: 浮動小数点演算のため、テストでは適切なtoleranceを使用（例: toBeCloseTo）
- **原単位の出典**: 冷房/暖房負荷原単位は一般的な設計基準値。将来的にユーザーカスタマイズ可能にする
- **requestIdleCallbackのテスト**: テスト環境ではpolyfillまたはモックが必要
- **dirty nodeの重複計算防止**: 同一フレームで同じゾーンが複数回計算されないよう制御
