## 実装詳細

### 1. 🔵 PipeRouteToolコンポーネント作成 (REQ-1101)

**ファイル**: `packages/editor/src/components/tools/hvac/pipe-route-tool.tsx`

配管ルーティング用のツールコンポーネント。DuctRouteToolと同様のルーティングフローだが、配管固有の媒体・2管制約を持つ。

- ユーザー操作フロー: AHU冷温水ポートクリック → 折点クリック → 終端確定

- ルーティング中はプレビュー線をローカルstateで管理

### 2. 🔵 冷温水2管（CWS + CWR）(REQ-1105)

冷水往管と冷水還管をペアで描画する。

- AHUのchilled_water_supply（CHW_S）ポートからCWS配管を描画
- AHUのchilled_water_return（CHW_R）ポートからCWR配管を描画
- 2管は平行配置（オフセット距離を設定）
- 各PipeSegmentNode.medium にそれぞれ 'chilled_water_supply' / 'chilled_water_return' を設定

### 3. 🔵 PipeSizingSystemコンポーネント作成

**ファイル**: `packages/core/src/systems/hvac/pipe-sizing-system.tsx`

Coreシステムとして実装。配管ルーティング後にdirtyNodesを監視し、AHUコイル能力から口径を自動算出する。

### 4. 🔵 冷水流量計算

AHUのcoolingCapacityから冷水流量を算出する。

- **計算式**: `flowRate = coolingCapacity / (cp × ΔT)`
  - flowRate: 冷水流量 (m³/s)
  - coolingCapacity: AHU冷房能力 (kW = kJ/s)
  - cp: 水の比熱 = 4186 J/(kg·K) = 4.186 kJ/(kg·K)
  - ΔT: 冷水温度差 (K) — デフォルト5K
  - 水の密度 ≈ 1000 kg/m³ で体積変換
- `flowRate = coolingCapacity / (4.186 × ΔT × 1000)` (m³/s)

### 5. 🔵 口径選定 + 標準口径表スナップ (REQ-1103)

流量と流速から口径を算出し、標準口径にスナップする。

- **計算式**: `diameter = √(4 × flowRate / (π × velocity))`
  - velocity: 目標流速 (m/s) — 1.5 m/s（推奨中央値）
- 標準口径表: 15A, 20A, 25A, 32A, 40A, 50A, 65A, 80A, 100A, 125A, 150A, 200A, 250A, 300A
- 計算値以上の最小標準口径にスナップ

### 6. 🔵 流速1.0~2.0 m/s範囲制約 (REQ-1103)

選定口径での実流速が許容範囲内か検証する。

- `実流速 = flowRate / (π × (diameter/2)²)`
- 実流速 < 1.0 m/s: 口径を1サイズ下げて再検証（下限）
- 実流速 > 2.0 m/s: 口径を1サイズ上げて再検証（上限）
- 範囲内に収まるまで反復
- 制約を満たせない場合: 警告ログ出力 + 最も近い口径を採用

### 7. 🔵 等価長さ法で圧損概算 (REQ-1104)

配管の圧損を等価長さ法で概算する。

- **直管圧損**: ダルシー・ワイスバッハ式 `ΔP = λ × (L / d) × (ρ × v² / 2)`
  - λ: 0.02（鋼管概算値）
  - ρ: 1000 kg/m³（水）
- **継手等価長さ**: 実長 × 係数（一般的に直管長の50%を加算）
- `総等価長さ = 直管長 + 継手等価長さ`
- 総圧損 = 直管圧損式で総等価長さを使用

### 8. 🟡 流量ゼロ時のスキップ+警告 (EDGE-103)

AHUのcoolingCapacityが0の場合のエッジケース処理。

- coolingCapacity = 0 → flowRate = 0 → 口径計算をスキップ
- `system:warning` イベントで「冷房能力が設定されていないため配管口径を算出できません」警告
- 配管は描画されるが口径未設定状態（diameter=0）で表示

## 注意事項

- **Toolsルール**: ツールはThree.js APIを直接呼び出さない。@pascal-app/viewerからインポートしない
- **コアシステムルール**: pipe-sizing-system.tsxはpackages/core/src/systems/に配置。Three.jsインポート禁止
- **DuctRouteToolとの共通化**: ポートスナップ、ルーティングフローなど共通ロジックが多い。共通ユーティリティ関数への抽出を推奨
- **2管の平行配置**: CWSとCWRの平行オフセット距離はMVPでは固定値（例: 0.1m）。将来的にはユーザー設定可能に
- **温水配管**: MVPでは冷水のみ対応。温水はスキーマ上は対応可能だが、UIフローは将来タスク
- **単位**: 口径はA（呼び径）で表示。内部計算はm単位。A→m変換テーブルが必要
