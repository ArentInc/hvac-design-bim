## 実装詳細

### 1. 🔵 AhuRendererコンポーネント作成

`packages/viewer/src/components/renderers/hvac/ahu-renderer.tsx` にレンダラーコンポーネントを作成する。

```typescript
// packages/viewer/src/components/renderers/hvac/ahu-renderer.tsx
import { useRef } from 'react'
import type { Group } from 'three'
import { useRegistry } from '../../../hooks/use-registry'
import { useNodeEvents } from '../../../hooks/use-node-events'

interface AhuRendererProps {
  nodeId: string
  position: [number, number, number]
  dimensions: { width: number; depth: number; height: number }
  ports: AhuPort[]
}

export function AhuRenderer({ nodeId, position, dimensions, ports }: AhuRendererProps) {
  const ref = useRef<Group>(null)
  useRegistry(nodeId, 'ahu', ref)
  useNodeEvents('ahu', nodeId)

  return (
    <group ref={ref} position={position}>
      {/* AHU本体 + ポート */}
    </group>
  )
}
```

- viewerパッケージに配置（レンダラーはviewerの責務）
- editorやapps/editorからインポートしない（Viewer Isolationルール）
- ジオメトリ生成は行わない（Systemsの責務）

### 2. 🔵 直方体メッシュ (REQ-1504)

AHU本体をdimensionsに基づく直方体で表現する。

```tsx
<mesh>
  <boxGeometry args={[dimensions.width, dimensions.height, dimensions.depth]} />
  <meshStandardMaterial color="#607D8B" />
</mesh>
```

- width, depth, heightはAhuNodeのdimensionsから取得
- カラーはHVAC機器用の標準色（グレー系）
- SCENE_LAYER（レイヤー0）に配置

### 3. 🔵 ポート円筒メッシュ表示

各ポートをAHU本体からの相対位置に円筒メッシュで表示する。

```tsx
{ports.map(port => (
  <mesh key={port.id} position={port.relativePosition}>
    <cylinderGeometry args={[port.diameter / 2, port.diameter / 2, 0.1, 16]} />
    <meshStandardMaterial color={getPortColor(port.type)} />
  </mesh>
))}
```

- ポートタイプ（SA, RA, CWS, CWR等）に応じた色分け
  - SA（給気）: 青系
  - RA（還気）: 赤系
  - CWS（冷水供給）: 水色
  - CWR（冷水還り）: 緑系
- ポート径はカタログ値から取得

### 4. 🔵 useRegistry登録

sceneRegistryにAHUメッシュを登録する。

```typescript
useRegistry(nodeId, 'ahu', ref)
```

- useRegistryフックがuseLayoutEffectでsceneRegistry.nodesとsceneRegistry.byType.ahuに登録
- アンマウント時に自動的にクリーンアップ

### 5. 🔵 useNodeEventsイベント発火

イベントバスにAHU関連イベントを発火する。

```typescript
useNodeEvents('ahu', nodeId)
```

- `ahu:click`, `ahu:enter`, `ahu:leave` 等のイベントを発火
- レンダラーはイベント発火のみ行い、リスンしない（イベントバスルール）

### 6. 🟡 ポートラベル表示

各ポートにタイプ名（SA, RA等）のラベルを表示する。

```tsx
{ports.map(port => (
  <group key={port.id} position={port.relativePosition}>
    {/* 円筒メッシュ */}
    <Html center distanceFactor={5}>
      <span style={{ fontSize: '10px', color: 'white' }}>
        {port.label}
      </span>
    </Html>
  </group>
))}
```

- drei/HtmlコンポーネントまたはText3Dでラベル表示
- カメラ距離に応じたスケーリングで視認性確保
- 優先度は低め（基本的な3D表示が先）

## 注意事項

- **Viewer Isolationルール**: editorパッケージやuseEditorストアからインポートしない。「読み取り専用ビューアでも意味があるか？」のテストに合格すること
- **ジオメトリ生成はSystemsの責務**: レンダラーはプレースホルダーメッシュを作成し、Systemsが生成したジオメトリを適用する
- **Three.jsレイヤー**: SCENE_LAYER = 0に配置。EDITOR_LAYERやZONE_LAYERには配置しない
- **パフォーマンス**: ポート数が多い場合のメッシュ数に注意。InstancedMeshの使用を検討
