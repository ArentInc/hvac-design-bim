## 実装詳細

### 1. 🔵 sample-architecture.jsonの読込ロジック (REQ-102)

建築参照JSONファイルを読み込み、バリデーション後にストアへ保持する。

**ファイル**: `packages/core/src/loaders/architecture-loader.ts`

```typescript
import { z } from 'zod'

// 建築参照JSONのスキーマ定義
const ArchitectureWallSchema = z.object({
  id: z.string(),
  orientation: z.enum(['N', 'NE', 'E', 'SE', 'S', 'SW', 'W', 'NW']),
  vertices: z.array(z.object({ x: z.number(), y: z.number(), z: z.number() })),
  wallArea: z.number().positive(),
  glazingRatio: z.number().min(0).max(1),
})

const ArchitectureLevelSchema = z.object({
  levelId: z.string(),
  floorHeight: z.number(),
  ceilingHeight: z.number(),
  floorOutline: z.array(z.object({ x: z.number(), y: z.number() })),
  externalWalls: z.array(ArchitectureWallSchema),
})

const ArchitectureSchema = z.object({
  buildingName: z.string(),
  levels: z.array(ArchitectureLevelSchema),
})

export type Architecture = z.infer<typeof ArchitectureSchema>
export type ArchitectureLevel = z.infer<typeof ArchitectureLevelSchema>
export type ArchitectureWall = z.infer<typeof ArchitectureWallSchema>

export function parseArchitecture(json: unknown): Architecture {
  return ArchitectureSchema.parse(json)
}
```

- Zodスキーマによるバリデーション
- 不正データ時はZodErrorをthrow
- パース結果はuseSceneまたは専用ストアに保持

### 3. 🔵 外壁面の方位とガラス面積比メタデータ保持 (REQ-103)

外壁面データから方位（orientation）とガラス面積比（glazingRatio）を抽出し、後続の負荷計算で参照可能にする。

**ファイル**: `packages/core/src/loaders/architecture-metadata.ts`

```typescript
export interface WallMetadata {
  wallId: string
  orientation: 'N' | 'NE' | 'E' | 'SE' | 'S' | 'SW' | 'W' | 'NW'
  wallArea: number
  glazingRatio: number
  vertices: { x: number; y: number; z: number }[]
}

export function extractWallMetadata(level: ArchitectureLevel): WallMetadata[] {
  return level.externalWalls.map((wall) => ({
    wallId: wall.id,
    orientation: wall.orientation,
    wallArea: wall.wallArea,
    glazingRatio: wall.glazingRatio,
    vertices: wall.vertices,
  }))
}
```

- レベルごとの外壁面メタデータ抽出
- TASK-0014（PerimeterEditTool）およびTASK-0015（LoadCalcSystem）で参照

## 注意事項

- **ビューア分離**: architecture-reference-renderer.tsxはpackages/viewerに配置するが、editorからの依存は持たないこと。建築参照データはprops経由で受け渡す
- **パフォーマンス**: 大規模建築JSONの場合、パース処理が重くなる可能性がある。非同期読込を検討
- **座標系**: 建築参照JSONの座標系がビューアの座標系と一致していることを前提とする。不一致の場合の変換は将来課題
- **サンプルデータ**: 開発用のsample-architecture.jsonをリポジトリに含める
