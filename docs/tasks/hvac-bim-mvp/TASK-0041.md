## 実装詳細

### 1. 🔵 ゾーン条件変更カスケード (REQ-1801)

ゾーンの条件（面積、用途、方位等）が変更された場合の連鎖再計算。

- **トリガー**: HvacZoneNodeの条件パラメータ変更 → markDirty
- **カスケード順序**:
  1. 負荷計算（LoadCalculationSystem）再実行 → calcResult更新
  2. 系統集計（SystemGroupingの負荷合算）再計算
  3. 機器候補更新（EquipmentSelectionSystem）
  4. 風量再配分（AirflowDistributionSystem）
  5. ダクト寸法再計算（DuctSizingSystem）
  6. 圧損再計算（PressureDropSystem）
  7. 警告再生成（ValidationSystem）
- **検証**: 各ステップでcalcResultが更新され、最終的に警告が再評価される

### 2. 🔵 グルーピング変更カスケード (REQ-1802)

系統グルーピング（ゾーンの系統割当）が変更された場合の連鎖再計算。

- **トリガー**: ゾーンの系統割当変更 → markDirty
- **カスケード順序**:
  1. 系統集計再計算（合算負荷の変更）
  2. 機器候補更新（合算負荷に基づく再選定）
- **検証**: 系統の合算負荷が正しく再計算され、機器候補が更新される

### 3. 🔵 ルート変更カスケード (REQ-1803)

ダクトルートまたは配管ルートが変更された場合の連鎖再計算。

- **トリガー**: ダクト区間の追加/削除/変更、配管の追加/削除/変更 → markDirty
- **カスケード順序**:
  1. 風量再配分（トポロジー変更に伴う再配分）
  2. ダクト寸法再計算
  3. 圧損再計算
  4. 警告再生成
- **検証**: 新しいルートに基づく風量・寸法・圧損が正しく計算される

### 4. 🔵 機器変更カスケード (REQ-1804)

機器（AHU等）が変更された場合の連鎖再計算。

- **トリガー**: 機器ノードの変更（機種変更、能力変更等） → markDirty
- **カスケード順序**:
  1. 接続風量の再計算（機器能力に基づく）
  2. ポート整合性チェック
  3. ダクト/配管再計算
- **検証**: 機器変更後に接続ダクト・配管の計算結果が更新される

### 5. 🔵 dirtyマーク連鎖伝播の検証

各カスケードにおいてdirtyマークが正しく連鎖伝播することを検証する。

- ノードAのdirty → SystemAが処理 → ノードBをdirtyマーク → SystemBが処理 → ...
- 全てのdirtyマークが最終的にクリアされる
- 循環参照によるdirtyマーク無限ループが発生しない

### 6. 🟡 計算完了までの時間計測（NFR-002: 5秒以内目標）

再計算カスケード全体の実行時間を計測し、NFR-002の目標（5秒以内）を確認する。

- 各カスケードの開始〜完了時間を計測
- 最も重いゾーン条件変更カスケードで5秒以内を目標
- ボトルネックの特定と改善ポイントの記録

## 単体テスト要件

**テストファイル**: `packages/core/src/systems/hvac/__tests__/recalculation-cascade.test.ts`

### テスト1: ゾーン条件変更→負荷再計算
- **信頼性**: 🔵
- **Given**: 計算済みのゾーン（面積100m²、事務室）
- **When**: ゾーンの面積を200m²に変更
- **Then**: 負荷計算結果が更新され、冷房負荷が約2倍になる

### テスト2: ゾーン条件変更→警告再生成
- **信頼性**: 🔵
- **Given**: 警告なしの完成状態のシーン
- **When**: ゾーン条件を変更し風速超過が発生する状態にする
- **Then**: velocity_exceeded警告が新たに生成される

### テスト3: グルーピング変更→機器候補更新
- **信頼性**: 🔵
- **Given**: 系統Aに2ゾーン割当済み
- **When**: 3つ目のゾーンを系統Aに追加
- **Then**: 系統Aの合算負荷が増加し、機器候補が更新される

### テスト4: ルート変更→寸法再計算
- **信頼性**: 🔵
- **Given**: 計算済みのダクトルート
- **When**: ダクト区間を1本追加
- **Then**: 風量再配分→寸法再計算が実行される

### テスト5: 機器変更→接続ダクト再計算
- **信頼性**: 🔵
- **Given**: AHUに接続されたダクトルート
- **When**: AHUの風量能力を変更
- **Then**: 接続ダクトの風量・寸法が再計算される

### テスト6: dirtyマーク完全クリア確認
- **信頼性**: 🔵
- **Given**: 任意のカスケードトリガー
- **When**: カスケードが完了
- **Then**: dirtyNodesが空になる

### テスト7: 計算時間計測
- **信頼性**: 🟡
- **Given**: 3ゾーン・1系統・10ダクト区間のシーン
- **When**: ゾーン条件変更カスケードを実行
- **Then**: 計算完了まで5秒以内

## 注意事項

- **カスケード順序**: Systemの実行順序はuseFrameの呼び出し順に依存する。必要に応じて優先度制御を実装
- **循環防止**: dirty→計算→dirty→... の無限ループを防止するガード条件を設ける
- **テスト安定性**: 計算結果の数値比較は適切な許容誤差（epsilon）を設定
- **Coreパッケージ**: カスケード制御ロジックはcoreパッケージに配置。viewer/editorへの依存は不可
