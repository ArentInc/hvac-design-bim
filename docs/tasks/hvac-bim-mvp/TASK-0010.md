### 1. 🔵 モード切替UIコンポーネント

**ファイル**: `packages/editor/src/components/ui/mode-switcher.tsx`

REQ-1401に基づく。上部バーに配置する建築/HVAC切替ボタン。

```typescript
export function ModeSwitcher() {
  const { editorMode, setEditorMode } = useEditor()

  return (
    <div className="mode-switcher">
      <button
        data-active={editorMode === 'architecture'}
        onClick={() => setEditorMode('architecture')}
      >
        建築
      </button>
      <button
        data-active={editorMode === 'hvac'}
        onClick={() => setEditorMode('hvac')}
      >
        HVAC
      </button>
    </div>
  )
}
```

### 3. 🔵 NodeRenderer 7ノード分岐追加

**ファイル**: `packages/viewer/src/components/renderers/node-renderer.tsx`

既存の NodeRenderer コンポーネントに7HVACノードタイプの条件分岐を追加。

```typescript
function NodeRenderer({ node }: { node: AnyNode }) {
  switch (node.type) {
    // 既存ノード...
    case 'wall': return <WallRenderer node={node} />
    // ...

    // HVAC ノード追加
    case 'hvac_zone': return <HvacZoneRenderer node={node} />
    case 'system': return null  // SystemNodeは3D表現なし
    case 'ahu': return <AhuRenderer node={node} />
    case 'diffuser': return <DiffuserRenderer node={node} />
    case 'duct_segment': return <DuctSegmentRenderer node={node} />
    case 'duct_fitting': return <DuctFittingRenderer node={node} />
    case 'pipe_segment': return <PipeSegmentRenderer node={node} />

    default: return null
  }
}
```

**重要**: レンダラーコンポーネント（HvacZoneRenderer等）はまだ実装されていないため、プレースホルダー（null返却 or 仮メッシュ）を配置する。

```typescript
// プレースホルダーレンダラー例
function HvacZonePlaceholder({ node }: { node: HvacZoneNode }) {
  return null  // 後続タスクで実装
}
```

### 4. 🟡 モード切替時の建築要素参照表示化ロジック

HVACモード切替時に建築要素を半透明参照表示にするロジック。

- editorMode === 'hvac' 時:
  - 建築ノード（Wall, Slab等）の opacity を下げる（0.3程度）
  - 建築ノードのインタラクション（click等）を無効化
  - HVACノードのみインタラクティブ

- editorMode === 'architecture' 時:
  - HVACノードの表示/非表示制御（薄く表示 or 非表示）

実装方針: Three.js layers または material.opacity の動的変更

## 注意事項

- **Viewer Isolation**: NodeRendererは `@pascal-app/viewer` パッケージに属する。HVACプレースホルダーレンダラーもviewerパッケージ内に配置。editor パッケージからのインポートは禁止（CLAUDE.md Critical Rule）
- **ModeSwitcherはeditorパッケージ**: UIコントロール（ModeSwitcher, フェーズタブ）は `@pascal-app/editor` パッケージに配置
- **プレースホルダー戦略**: レンダラーが未実装のノードタイプは `null` を返す。コンソールwarningは出さない（開発中のノイズになるため）
- **SystemNodeの3D表現**: SystemNodeは論理的なグルーピングノードであり、3D表現は持たない。NodeRendererではnullを返す
- **参照表示化ロジック（🟡）**: Three.js layersの使用可否は既存レイヤー設定（SCENE_LAYER=0, EDITOR_LAYER=1, ZONE_LAYER=2）との整合性を確認してから決定。material.opacity変更の方が安全な可能性
