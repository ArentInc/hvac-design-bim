## 実装詳細

### 1. 🔵 DuctVisualSystemコンポーネント作成

**ファイル**: `packages/viewer/src/systems/hvac/duct-visual-system.tsx`

Viewerシステムとして実装。useFrame内でdirtyNodesを監視し、ダクトメッシュの視覚プロパティを更新する。

- dirtyNodesにDuctSegmentが含まれる場合にビジュアル更新トリガー
- sceneRegistry.nodes から対応するObject3Dを取得して更新

### 2. 🔵 断面比例太さ更新（width/height反映）

DuctSegmentNodeのwidth/heightに基づいてメッシュスケールを更新する。

- メッシュのscale.x = width（幅方向）
- メッシュのscale.y = height（高さ方向）
- 長さ（Z方向スケール）はstart→end距離で変化なし
- 寸法未確定（width=0 || height=0）の場合は最小太さで表示

### 3. 🔵 接続状態・寸法確定状態による色変更 (REQ-1502)

ダクトの状態に応じたマテリアルカラー変更。

- **寸法確定+両端接続済み**: #90CAF9（水色） — 正常状態
- **寸法確定+片端未接続**: #FFA726（オレンジ） — 接続不完全
- **寸法未確定**: #BDBDBD（グレー）破線 — サイジング待ち
- **エラー状態（圧損超過等）**: #EF5350（赤）
- MeshStandardMaterial.color を直接更新

### 4. 🔵 風量ラベル + 寸法ラベル表示 (REQ-1502)

ダクトメッシュの上方に風量と寸法の情報ラベルを表示する。

- **風量ラベル**: `"600 m³/h"` 形式
- **寸法ラベル**: `"400×300"` 形式（mm単位）
- HTML要素またはSprite + CanvasTextureで実装
- ダクトメッシュの中点上方にオフセット配置
- カメラ距離に応じたラベルサイズ調整（遠距離で非表示）

### 5. 🔵 sceneRegistry経由でメッシュ参照・更新

sceneRegistryを読み取り専用で使用してメッシュを更新する。

- `sceneRegistry.nodes.get(nodeId)` でObject3D参照取得
- `sceneRegistry.byType.duct_segment` で全ダクトセグメントID取得
- Object3Dのchildren/material/scaleを直接操作
- レンダラーが管理するメッシュの視覚プロパティのみ変更（構造変更は行わない）

## 注意事項

- **Viewer Isolation**: packages/viewerはpackages/editorに依存してはならない。useEditorやphase/mode参照は禁止
- **パフォーマンス**: useFrame内での更新はdirtyNodesに含まれるノードのみ処理。全ダクトの毎フレーム更新は避ける
- **ラベル描画**: HTML overlayまたはSprite方式。MVP段階ではシンプルなSprite + CanvasTextureを推奨
- **色定数**: カラーコードは定数ファイルに外部定義し、テーマ対応の余地を残す
- **sceneRegistryの読み取り**: システムはsceneRegistryの読み取り専用コンシューマー。ノード登録/解除は行わない
