## 実装詳細

### 1. 🔵 HvacZoneNode Zod スキーマ定義

**ファイル**: `packages/core/src/schema/nodes/hvac-zone.ts`

interfaces.ts の HvacZoneNode 型定義と REQ-201~209 に基づき、以下のフィールドを持つ Zod スキーマを定義する。

- **id prefix**: `'hvac_zone'`
- **type**: `'hvac_zone'`
- **基本フィールド**:
  - `zoneName`: string — ゾーン名称
  - `usage`: ZoneUsage enum — 用途区分
  - `floorArea`: number — 床面積 (m²)
  - `ceilingHeight`: number (default: 2.7) — 天井高 (m)
  - `occupantDensity`: number (default: 0.15) — 在室密度 (人/m²)
  - `boundary`: `[number, number][]` — ゾーン境界ポリゴン座標配列
  - `designConditions`: DesignConditions — 設計条件
  - `perimeterSegments`: PerimeterSegment[] — ペリメータセグメント配列
  - `systemId`: string | null — 所属系統ID
  - `calcResult`: HvacZoneCalcResult | null — 計算結果（初期null）

- **DesignConditions サブスキーマ**:
  - `coolingSetpoint`: number (default: 26) — 冷房設定温度 (°C)
  - `heatingSetpoint`: number (default: 22) — 暖房設定温度 (°C)
  - `relativeHumidity`: number (default: 50) — 相対湿度 (%)
  - `supplyAirTempDiff`: number (default: 10) — 給気温度差 (°C)

- **PerimeterSegment サブスキーマ**:
  - `orientation`: Orientation enum — 方位
  - `wallArea`: number — 壁面積 (m²)
  - `glazingRatio`: number (0.0~1.0) — 窓面積比率

### 2. 🔵 SystemNode Zod スキーマ定義

**ファイル**: `packages/core/src/schema/nodes/system.ts`

interfaces.ts の SystemNode 型定義と REQ-401~405 に基づく。

- **id prefix**: `'system'`
- **type**: `'system'`
- **フィールド**:
  - `systemName`: string — 系統名称
  - `servedZoneIds`: string[] — 管轄ゾーンID配列
  - `ahuId`: string | null — 割当AHU ID
  - `aggregatedLoad`: AggregatedLoad | null — 集計負荷（初期null）
  - `status`: SystemStatus enum — 系統ステータス

- **AggregatedLoad サブスキーマ**:
  - `totalCoolingLoad`: number — 合計冷房負荷 (W)
  - `totalHeatingLoad`: number — 合計暖房負荷 (W)
  - `totalAirflow`: number — 合計風量 (m³/h)

- **SystemStatus enum**: `'draft' | 'equipment_selected' | 'routed' | 'calculated' | 'validated'`

### 3. 🔵 共有型定義

**ファイル**: `packages/core/src/schema/nodes/hvac-shared.ts`（または各ファイル内）

- **Orientation enum**: `'N' | 'NE' | 'E' | 'SE' | 'S' | 'SW' | 'W' | 'NW'`
- **ZoneUsage enum**: `'office_general' | 'office_server' | 'conference' | 'reception' | 'corridor'`
- **PortMedium enum**: `'supply_air' | 'return_air' | 'chilled_water' | 'hot_water' | 'refrigerant'`

## 注意事項

- **既存パターン準拠**: BaseNode.extend() パターンを使用すること。packages/core/src/schema/nodes/ 内の既存スキーマ（wall.ts等）を参考にする
- **ID自動生成**: NodeType.parse() 時にID prefixが自動付与される既存の仕組みに従う
- **calcResult初期値**: HvacZoneCalcResult は初期状態ではnull。計算実行後にシステムが値を設定する
- **boundary座標**: 2D座標配列（[x, z]）。3Dのy座標はLevelのelevationから導出
- **AnyNode統合はTASK-0005**: 本タスクではスキーマ定義のみ。AnyNode unionへの追加は別タスク
