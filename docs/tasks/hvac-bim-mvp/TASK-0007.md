## 実装詳細

### 1. 🔵 EditorMode 型定義

**ファイル**: `packages/editor/src/store/use-editor.tsx`

```typescript
type EditorMode = 'architecture' | 'hvac'
```

useEditorストアの state に `editorMode: EditorMode` フィールドを追加（初期値: `'architecture'`）。

### 2. 🔵 Phase 型拡張

既存の Phase 型を拡張:

```typescript
// 既存
type ArchitecturePhase = 'site' | 'structure' | 'furnish'

// 追加
type HvacPhase = 'zone' | 'equip' | 'route' | 'calc'

// 統合
type Phase = ArchitecturePhase | HvacPhase
```

### 3. 🔵 HvacTool 型定義

interfaces.ts の HvacTool に基づく10ツール:

```typescript
type HvacTool =
  | 'zone_draw'        // ゾーン描画
  | 'zone_edit'        // ゾーン編集
  | 'ahu_place'        // AHU配置
  | 'diffuser_place'   // 制気口配置
  | 'duct_route'       // ダクトルーティング
  | 'duct_edit'        // ダクト編集
  | 'pipe_route'       // 配管ルーティング
  | 'pipe_edit'        // 配管編集
  | 'system_assign'    // 系統割当
  | 'calc_run'         // 計算実行
```

### 4. 🔵 phasesByEditorMode マッピング

```typescript
const phasesByEditorMode: Record<EditorMode, Phase[]> = {
  architecture: ['site', 'structure', 'furnish'],
  hvac: ['zone', 'equip', 'route', 'calc'],
}
```

### 5. 🔵 toolsByPhase マッピング

```typescript
const toolsByPhase: Record<HvacPhase, HvacTool[]> = {
  zone: ['zone_draw', 'zone_edit'],
  equip: ['ahu_place', 'diffuser_place', 'system_assign'],
  route: ['duct_route', 'duct_edit', 'pipe_route', 'pipe_edit'],
  calc: ['calc_run'],
}
```

### 6. 🔵 setEditorMode メソッド

```typescript
setEditorMode: (mode: EditorMode) => {
  const defaultPhase = mode === 'hvac' ? 'zone' : 'site'
  set({
    editorMode: mode,
    phase: defaultPhase,
    mode: 'select',       // モード切替時はselectにリセット
    activeTool: null,     // ツールもリセット
  })
}
```

### 7. 🔵 既存setPhase/setModeのeditorModeフィルタリング

setPhase メソッドにバリデーション追加:

```typescript
setPhase: (phase: Phase) => {
  const { editorMode } = get()
  const validPhases = phasesByEditorMode[editorMode]
  if (!validPhases.includes(phase)) return // 無効なフェーズは無視
  set({ phase, mode: 'select', activeTool: null })
}
```

## 注意事項

- **既存動作への影響最小化**: 既存の Phase 型 ('site' | 'structure' | 'furnish') とそれに依存するコード全てに影響がないことを確認。EditorMode のデフォルト値 'architecture' により、既存動作は変更なし
- **Zustand 5 パターン**: useEditor は Zustand 5 で実装されている。setState のパターンに従うこと
- **activeTool の型**: 既存の activeTool 型と HvacTool 型の union を適切に管理。型の後方互換性を維持
- **Zundo（undo/redo）**: useEditor は Zundo を使用していない（useScene のみ）。モード切替のundo/redoは不要
- **テストでのストアリセット**: 各テストケース前に useEditor.setState() でストアをリセットすること
