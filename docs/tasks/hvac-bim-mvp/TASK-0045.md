## 実装詳細

### 1. 🔵 風量自動配分完了→全区間風量ラベル一斉更新 (REQ-1603)

AirflowDistributionSystemが風量自動配分を完了した際に、3Dビュー上の全ダクト区間の風量ラベルが即座に更新されるフィードバック。

- AirflowDistributionSystem完了時にイベントバス経由で通知
- DuctVisualSystem（TASK-0033）がイベントを受信し、全ダクトラベルを更新
- ラベル更新時に短いアニメーション（フラッシュ効果等）で変更を視覚化
- 更新されたラベルの数をコンソールまたはステータスバーに表示

### 2. 🔵 保存完了→トースト通知 (REQ-1604)

IndexedDBへの保存完了時にトースト通知を表示する。

**ファイル**: `packages/editor/src/components/ui/toast/toast-provider.tsx`

- 保存完了イベントを検知してトースト表示
- トーストメッセージ: 「プロジェクトを保存しました」
- 表示位置: 画面右下
- 自動消去: 3秒後にフェードアウト
- 複数トーストのスタック表示対応

**ファイル**: `packages/editor/src/components/ui/toast/use-toast.ts`

- useToast() フック: toast({ message, type, duration }) でトースト表示
- type: 'success' | 'error' | 'info' | 'warning'
- トースト状態管理（表示中のトースト配列）

### 3. 🟡 計算進捗インジケータ表示

再計算カスケード実行中の進捗表示。

- 計算開始時にインジケータ表示（スピナーまたはプログレスバー）
- 計算完了時にインジケータ非表示
- 表示位置: ツールバーまたはステータスバー
- 長時間計算（1秒以上）の場合のみ表示

## UI/UX要件

- トースト通知はユーザーの操作を妨げない位置に表示（右下推奨）
- 成功トースト: 緑系アクセントカラー
- エラートースト: 赤系アクセントカラー
- 風量ラベル更新時のフラッシュ効果は控えめ（0.3秒程度のハイライト）
- 計算進捗インジケータは小さく目立たない位置に配置
- アニメーションはprefers-reduced-motionメディアクエリに対応

## 注意事項

- **editorパッケージ**: トースト通知はeditorパッケージに配置。viewerパッケージではトースト表示しない
- **イベントバス**: 風量配分完了通知はイベントバス経由。レンダラーがリスンしてはならないルールに注意（SystemがSprite更新）
- **パフォーマンス**: ラベル一斉更新時のフレーム落ちに注意。必要に応じてバッチ更新
- **アクセシビリティ**: トースト通知にaria-live="polite"を設定し、スクリーンリーダーに対応
