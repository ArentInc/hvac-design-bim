### 3. 🔵 AhuNode作成とcreateNode

配置位置クリック時にAhuNodeを作成する。

```typescript
const handlePlaceAhu = (position: [number, number, number]) => {
  if (!selectedCatalogEntry) return

  const { createNode, updateNode } = useScene.getState()
  const { levelId } = useViewer.getState()

  const ahuNode = AhuNode.parse({
    equipmentName: selectedCatalogEntry.name,
    position,
    dimensions: {
      width: selectedCatalogEntry.width,
      depth: selectedCatalogEntry.depth,
      height: selectedCatalogEntry.height,
    },
    ports: transformPorts(selectedCatalogEntry.ports, position),
    coolingCapacity: selectedCatalogEntry.coolingCapacity,
    heatingCapacity: selectedCatalogEntry.heatingCapacity,
    ratedAirflow: selectedCatalogEntry.ratedAirflow,
  })

  createNode(ahuNode, levelId)
}
```

- AhuNode.parse()でZodバリデーション（IDは自動生成: `ahu_xxx`）
- createNode(node, levelId)でレベルの子ノードとして作成
- カタログのスペックデータをAhuNodeにコピー

### 4. 🔵 SystemNodeのahuId更新

配置したAHUを系統に紐付ける。

```typescript
const handlePlaceAhu = (position: [number, number, number]) => {

```

- 系統コンテキストがある場合、SystemNodeのahuIdを更新
- 系統に紐付かないスタンドアロン配置も許可

### 5. 🔵 ポート座標のカタログ定義からの変換

カタログ定義のポート相対座標をワールド座標に変換する。

```typescript
const transformPorts = (
  catalogPorts: CatalogPort[],
  ahuPosition: [number, number, number]
): AhuPort[] => {
  return catalogPorts.map(port => ({
    id: generatePortId(),
    type: port.type,      // 'SA' | 'RA' | 'CWS' | 'CWR' | ...
    label: port.label,
    diameter: port.diameter,
    relativePosition: port.relativePosition,  // AHU本体からの相対座標
    connectedTo: null,
  }))
}
```

- カタログのポート位置はAHU本体の原点基準の相対座標
- AhuRendererがrelativePositionを使用して表示
- ダクト/パイプ接続時にワールド座標への変換が必要

## 注意事項

- **Toolsルール遵守**: Three.js APIを直接呼ばない。プレビュー表示はローカルステートで管理
- **viewerからインポートしない**: ツールはeditorパッケージの責務
- **undo/redo**: createNodeとupdateNodeはZundoの対象。配置操作は2つのノード操作（createNode + updateNode）を含むため、バッチ処理の検討が必要
- **座標系**: ポジションはワールド座標系。レベルのY座標を考慮
