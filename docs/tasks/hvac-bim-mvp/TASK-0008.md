## タスク概要

既存のmittイベントバスに7種HVACノードのイベント型を追加する。既存パターン（`<nodeType>:<suffix>` 形式）に従い、hvac_zone, system, ahu, diffuser, duct_segment, duct_fitting, pipe_segment の各ノードタイプに対してclick, enter, leave等の標準イベントを定義する。レンダラー（後続タスク）がイベント発行に使用する。

## 実装詳細

### 1. 🔵 EditorEvents 型に HVACノードイベントを追加

**ファイル**: `packages/core/src/events/bus.ts`（または該当するイベント型定義ファイル）

既存の NodeEvents ジェネリック型パターンを使用して、7ノードタイプのイベントを追加:

```typescript
// 既存パターン（参考）
type NodeEvents<T extends string> = {
  [K in `${T}:click`]: { nodeId: string; event: PointerEvent }
} & {
  [K in `${T}:enter`]: { nodeId: string; event: PointerEvent }
} & {
  [K in `${T}:leave`]: { nodeId: string; event: PointerEvent }
}

// EditorEvents に追加
type EditorEvents = {
  // 既存イベント...
  // ...

  // HVAC ノードイベント追加
} & NodeEvents<'hvac_zone'>
  & NodeEvents<'system'>
  & NodeEvents<'ahu'>
  & NodeEvents<'diffuser'>
  & NodeEvents<'duct_segment'>
  & NodeEvents<'duct_fitting'>
  & NodeEvents<'pipe_segment'>
```

### 2. 🔵 各イベントの型定義

各HVACノードタイプに以下の標準イベントを定義:

| イベント名 | ペイロード | 用途 |
|-----------|-----------|------|
| `{type}:click` | `{ nodeId: string; event: PointerEvent }` | ノードクリック |
| `{type}:enter` | `{ nodeId: string; event: PointerEvent }` | ポインター侵入 |
| `{type}:leave` | `{ nodeId: string; event: PointerEvent }` | ポインター離脱 |

合計: 7タイプ x 3イベント = 21イベント

注意: 既存パターンに他のイベント（doubleclick, contextmenu等）がある場合はそれらも追加する。

## 注意事項

- **CLAUDE.md準拠**: 「レンダラーがイベントを発行（useNodeEvents経由）、リスナーは購読のみ。Grid eventsは空スペースインタラクション時に発火。リスナーはuseEffect内で同一関数参照でクリーンアップ」
- **mitt の型パラメータ**: mitt は `mitt<EditorEvents>()` の形式で型パラメータを受け取る。EditorEvents の型拡張がmittインスタンスに反映されることを確認
- **イベント名のアンダースコア**: `duct_segment:click` のようにノードタイプにアンダースコアが含まれる。既存パターンと整合しているか確認
- **パフォーマンス**: 21イベント追加による mitt のパフォーマンス影響は無視可能（イベント登録は遅延バインディング）
- **工数4h**: 既存パターンの拡張のみで新規設計不要。型定義の追加が主な作業
