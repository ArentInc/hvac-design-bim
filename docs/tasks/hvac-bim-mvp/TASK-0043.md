## タスク概要

HVACノードを含むシーン全体のIndexedDB永続化と復元が正しく動作することを検証する。既存のidb-keyvalベースの保存フローにHVACノードが含まれること、Zodスキーマ検証が全HVACノードをカバーすること、再読込後にcalcResultが復元され再計算で同じ結果が得られることを確認する。

## 実装詳細

### 1. 🔵 既存IndexedDB保存フローのHVACノード対応確認 (REQ-1301)

既存のidb-keyvalベースの永続化フローが、HVACノード型を正しくシリアライズ/デシリアライズすることを確認する。

- useSceneのpersistミドルウェア（Zundo + IndexedDB）がHVACノードを含むnodes Recordを保存
- HvacZoneNode, EquipmentNode, DiffuserNode, DuctSegmentNode, PipeSegmentNode等の全型が保存対象
- calcResult（計算結果オブジェクト）もノードデータの一部として保存
- 保存タイミング: ノード変更時の自動保存（debounce付き）

### 2. 🔵 Zodスキーマ検証の全HVACノードカバー (NFR-101)

IndexedDBから読込んだデータに対するZodスキーマバリデーションが全HVACノード型で正しく機能することを確認する。

- AnyNode union型にHVACノード型が全て含まれている（TASK-0005で実装済み）
- 不正なデータが読み込まれた場合にZodバリデーションエラーが適切に発生
- バリデーションエラー時のフォールバック動作（該当ノードのスキップまたはデフォルト値適用）

### 3. 🔵 再読込後のcalcResult復元 (REQ-1302)

ブラウザリロード後にcalcResult（計算結果）が保存データから復元されることを確認する。

- calcResultフィールドがシリアライズ可能な構造であること
- Date型やMap型等のシリアライズ不可な型が含まれていないこと
- 復元後のcalcResultが保存前と同一の値であること

### 4. 🔵 再読込後のmarkDirty→再計算→結果一致検証

復元後に全ノードをmarkDirtyして再計算を実行し、保存前と同じ結果が得られることを確認する。

- 復元後に全HVACノードをmarkDirty
- 全計算システムが再実行される
- 再計算後のcalcResultが保存前の値と一致（数値は許容誤差内）
- 警告配列が保存前と同一

### 5. 🟡 計算失敗ハンドリング (NFR-102)

計算が失敗した場合のcalcResultへのerror状態格納。

- 計算失敗時にcalcResult.error フィールドにエラー情報を格納
- エラー状態のノードが保存・復元後も適切に表示される
- エラー状態からの再計算トリガーが可能

## 単体テスト要件

**テストファイル**: `packages/core/src/systems/hvac/__tests__/persistence.test.ts`

### テスト1: HVACノードのシリアライズ
- **信頼性**: 🔵
- **Given**: HvacZoneNode, EquipmentNode, DuctSegmentNodeを含むシーン
- **When**: JSON.stringify(nodes)を実行
- **Then**: エラーなくシリアライズされ、JSON.parseで復元できる

### テスト2: Zodスキーマバリデーション通過
- **信頼性**: 🔵
- **Given**: 保存→復元されたHVACノードデータ
- **When**: AnyNode.parse(nodeData)を各ノードに適用
- **Then**: 全ノードがバリデーションを通過する

### テスト3: calcResult復元の一致性
- **信頼性**: 🔵
- **Given**: calcResultが設定されたHvacZoneNode
- **When**: シリアライズ→デシリアライズ
- **Then**: calcResultの全フィールドが保存前と同一

### テスト4: 再計算結果の一致性
- **信頼性**: 🔵
- **Given**: 計算済みシーンを保存→復元
- **When**: 全ノードをmarkDirty→再計算実行
- **Then**: 再計算結果が保存前のcalcResultと一致（許容誤差±0.1%）

### テスト5: 不正データのZodバリデーションエラー
- **信頼性**: 🔵
- **Given**: 必須フィールドが欠損したHVACノードデータ
- **When**: AnyNode.parse(invalidData)を実行
- **Then**: ZodErrorがスローされる

### テスト6: 計算失敗状態の保存・復元
- **信頼性**: 🟡
- **Given**: calcResult.errorが設定されたノード
- **When**: シリアライズ→デシリアライズ
- **Then**: error状態が保持される

## 注意事項

- **シリアライズ互換性**: calcResult内にDate, Map, Set, Function等のシリアライズ不可な型を含めない
- **データマイグレーション**: スキーマ変更時の既存データとの互換性は将来課題。MVPではバージョン管理は不要
- **IndexedDBサイズ**: 大量ノードのシーンでIndexedDBストレージ制限に注意（通常は問題にならない）
- **自動保存タイミング**: debounceにより高頻度の変更が集約される。デフォルト1秒程度を想定
