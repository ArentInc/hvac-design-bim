## タスク概要

AhuNode（空気調和機）と DiffuserNode（制気口）の Zod スキーマを定義する。Port型の共通定義も含む。AHUはHVAC系統の中心機器であり、Diffuserはゾーンへの給気/還気口を表す。両者はダクト接続のエンドポイントとして機能し、Port型を介して接続される。

## 実装詳細

### 1. 🔵 Port型の共通定義

**ファイル**: `packages/core/src/schema/nodes/hvac-shared.ts`（TASK-0002で作成済みファイルに追加）

interfaces.ts の Port 型定義に基づく共通ポートスキーマ。

- `id`: string — ポート固有ID
- `label`: string — ポートラベル（例: 'SA', 'RA', 'CHW_S'）
- `medium`: PortMedium enum — 媒体種別
- `position`: `[number, number, number]` — 3D座標
- `direction`: `[number, number, number]` — 接続方向ベクトル
- `connectedSegmentId`: string | null — 接続先セグメントID

PortMedium enum（TASK-0002で定義済み）: `'supply_air' | 'return_air' | 'chilled_water' | 'hot_water' | 'refrigerant'`

### 2. 🔵 AhuNode Zod スキーマ定義

**ファイル**: `packages/core/src/schema/nodes/ahu.ts`

interfaces.ts の AhuNode 型定義と REQ-504 に基づく。

- **id prefix**: `'ahu'`
- **type**: `'ahu'`
- **フィールド**:
  - `tag`: string — 機器タグ名（例: 'AHU-1'）
  - `equipmentName`: string — 機器名称
  - `position`: `[number, number, number]` — 設置位置
  - `rotation`: `[number, number, number]` — 回転角度（Euler）
  - `dimensions`: AhuDimensions — 寸法
  - `ports`: Port[] — ポート配列（最低2: SA, RA）
  - `airflowRate`: number — 定格風量 (m³/h)
  - `coolingCapacity`: number — 冷房能力 (kW)
  - `heatingCapacity`: number — 暖房能力 (kW)
  - `staticPressure`: number — 静圧 (Pa)
  - `systemId`: string — 所属系統ID

- **AhuDimensions サブスキーマ**:
  - `width`: number — 幅 (m)
  - `height`: number — 高さ (m)
  - `depth`: number — 奥行 (m)

### 3. 🔵 DiffuserNode Zod スキーマ定義

**ファイル**: `packages/core/src/schema/nodes/diffuser.ts`

interfaces.ts の DiffuserNode 型定義と REQ-602~604 に基づく。

- **id prefix**: `'diffuser'`
- **type**: `'diffuser'`
- **フィールド**:
  - `tag`: string — 機器タグ名（例: 'SA-1-01'）
  - `subType`: DiffuserSubType enum — 制気口種別
  - `position`: `[number, number, number]` — 設置位置
  - `neckDiameter`: number — ネック径 (mm)
  - `airflowRate`: number — 設計風量 (m³/h)
  - `port`: Port — 接続ポート（単一）
  - `hostDuctId`: string | null — 接続先ダクトセグメントID
  - `systemId`: string — 所属系統ID
  - `zoneId`: string — 所属ゾーンID

- **DiffuserSubType enum**: `'anemostat' | 'line' | 'universal' | 'nozzle' | 'return_grille'`

## 注意事項

- **Port型の再利用**: Port型はAhuNode（複数ポート）、DiffuserNode（単一ポート）、DuctFittingNode（TASK-0004）で共用される。hvac-shared.ts での定義を推奨
- **ポート方向ベクトル**: direction は正規化ベクトル（長さ1）を想定するが、スキーマレベルでは正規化チェックは行わない（ジオメトリ生成時にシステムが処理）
- **AHUポートパターン**: 典型的なAHUは4ポート（SA給気, RA還気, CHW_S冷水往, CHW_R冷水還）だが、スキーマ上はポート数の制約はない
- **DiffuserのzoneId**: DiffuserはゾーンとダクトのI/Fとして機能するため、systemIdとzoneId両方を持つ
