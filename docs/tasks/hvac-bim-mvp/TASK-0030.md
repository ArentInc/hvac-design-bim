## 実装詳細

### 1. 🔵 DuctRouteToolコンポーネント作成

**ファイル**: `packages/editor/src/components/tools/hvac/duct-route-tool.tsx`

ダクトルーティング用のツールコンポーネント。useEditorのmode='build'、activeTool='duct_route'時にアクティブ化。

- ユーザー操作フロー: 起点ポートクリック → 折点クリック(0回以上) → 終点ポートスナップ確定
- ローカルstate: routePoints[], previewLine, snapTarget

### 2. 🔵 ポートスナップ検出（proximity based）(REQ-704)

マウスカーソル近傍のポートを検出しスナップする。

- sceneRegistry.byType から AHU/Diffuser ノードのポート位置を取得
- カーソル位置からの距離がスナップ閾値（例: 0.3m）以内のポートを検出
- 最近接ポートにスナップインジケータ表示
- スナップ対象は connectedSegmentId === null の未接続ポートのみ

### 3. 🔵 起点→折点→終点のルーティングフロー

クリック操作による段階的なルーティング。

- **起点クリック**: ポートスナップ検出 → startPortId設定 → ルーティング開始状態へ
- **折点クリック**: 空間上の任意点をクリック → routePointsに追加
- **終点スナップ確定**: ポートスナップ検出 → endPortId設定 → 確定処理
- **Escキー**: ルーティングキャンセル → ローカルstate初期化

### 4. 🔵 DuctSegmentNode作成 + ポート更新

確定時にDuctSegmentNodeを作成し、ポートの接続情報を更新する。

- routePoints配列の各区間について:
  - `DuctSegmentNode.parse({ start, end, startPortId, endPortId, medium })` → `createNode(node, parentId)`
  - medium は起点ポートの medium を継承
- 起点ポートの `connectedSegmentId` を最初のセグメントIDに更新
- 終点ポートの `connectedSegmentId` を最後のセグメントIDに更新
- 折点がある場合、中間セグメント間はポートIDなし（segment-to-segment接続）

### 5. 🔵 ポートのconnectedSegmentId更新

ダクト接続時にポートの接続状態を更新する。

- `updateNode(ahuId, { ports: updatedPorts })` で起点AHUのポート更新
- `updateNode(diffuserId, { port: updatedPort })` で終点Diffuserのポート更新
- 接続解除時は `connectedSegmentId = null` に戻す

### 6. 🔵 DuctFittingNode作成（T分岐操作時）(REQ-705)

既存ダクトセグメント上で分岐操作を行った場合、T分岐継手を自動作成する。

- 既存セグメント上でのクリックを検出
- 元セグメントを分割: 2つの新セグメント + T分岐ノード作成
- DuctFittingNode.parse({ fittingType: 'tee', ports: [3ポート] }) → createNode
- 3ポートの direction ベクトルを適切に設定

### 7. 🔵 PortMedium不整合チェック (EDGE-003)

起点と終点のPortMediumが一致するかチェックする。

- 起点ポートのmediumと終点ポートのmediumを比較
- 不一致時: ルーティング確定を阻止 + ユーザーに警告表示
- 例: supply_air ポートに return_air ポートを接続しようとした場合

### 8. 🟡 プレビュー線のローカル状態管理

ルーティング中のプレビュー表示。

- 確定前のルート線をローカルstateで管理
- Line + LineDashedMaterial でプレビュー描画
- カーソル移動に追従する動的な線分
- スナップ対象がある場合はスナップ先まで延長
- プレビューはEDITOR_LAYER（レイヤー1）に配置

## 注意事項

- **Toolsルール**: ツールはThree.js APIを直接呼び出さない。プレビュージオメトリはローカルstateのみ
- **Toolsルール**: ツールは@pascal-app/viewerからインポートしない
- **ポートスナップ閾値**: 閾値は定数化し、将来的にユーザー設定可能にする想定
- **T分岐**: 既存セグメント分割は複雑な操作。元セグメントのdeleteNode → 2新セグメントのcreateNode → FittingNodeのcreateNode の順で実行
- **Undo対応**: 複数ノード操作はトランザクション的に処理。undo時に全操作が一括で巻き戻される必要がある
