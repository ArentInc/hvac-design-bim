## 実装詳細

### 1. 🔵 AHUカタログデータ

**ファイル**: `packages/core/src/data/catalog-ahu.json`

REQ-505に基づくAHU機器カタログ。5機種を定義。

```json
[
  {
    "modelId": "AHU-S-2000",
    "modelName": "小型AHU 2000",
    "airflowRate": 2000,
    "coolingCapacity": 12.0,
    "heatingCapacity": 8.0,
    "staticPressure": 300,
    "dimensions": { "width": 1.2, "height": 1.0, "depth": 0.8 },
    "ports": [...]
  }
  // ... 5機種: 2000, 5000, 10000, 20000, 30000 m³/h
]
```

フィールド:
- `modelId`: モデル識別子
- `modelName`: 表示名
- `airflowRate`: 定格風量 (m³/h)
- `coolingCapacity`: 冷房能力 (kW)
- `heatingCapacity`: 暖房能力 (kW)
- `staticPressure`: 静圧 (Pa)
- `dimensions`: 外形寸法 (m)
- `ports`: 標準ポート配置テンプレート

## 注意事項

- **データの技術的正確性**: AHU能力値、ダクトサイズ、配管口径はJIS規格準拠の現実的な値を使用すること。設計実務で使われない値は避ける
- **単位の統一**: 空調能力はkW、風量はm³/h、寸法はm（建築）またはmm（ダクト/配管）、圧力はPa/kPa。各JSONにunit情報を含めることを推奨
- **型安全インポート**: JSONファイルはTypeScriptの `import` で型推論が効くように、`resolveJsonModule: true` が tsconfig に設定されていることを確認
- **拡張性**: カタログデータは将来的にユーザーカスタマイズ対応を想定。構造を変えやすいフラットな形式で定義
- **日射補正係数はTypeScript**: 定数として使用頻度が高く、型安全性が重要なためTypeScriptファイルで定義（JSONではなく）
