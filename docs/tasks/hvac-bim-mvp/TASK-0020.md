## タスク概要

複数のHvacZoneを1つのAHU系統（SystemNode）にグルーピングするツール。ユーザーがゾーンを複数選択し「系統に追加」操作を行うと、SystemNodeが作成され各ゾーンにsystemIdが設定される。1ゾーンが複数系統に属さないN:1制約バリデーションを実装し、グルーピング後はaggregatedLoadが即時更新される。

### 2. 🔵 ゾーン複数選択UI

チェックボックスまたはCtrl+クリックによるゾーン複数選択機能を実装する。

```typescript
const [selectedZoneIds, setSelectedZoneIds] = useState<string[]>([])

const handleZoneSelect = (zoneId: string, isMulti: boolean) => {
  if (isMulti) {
    setSelectedZoneIds(prev =>
      prev.includes(zoneId)
        ? prev.filter(id => id !== zoneId)
        : [...prev, zoneId]
    )
  } else {
    setSelectedZoneIds([zoneId])
  }
}
```

- Ctrl/Meta + クリックで複数選択
- 既存のSelectionManagerのselectedIdsと連携
- 選択中のゾーンをハイライト表示

### 3. 🔵 系統作成とゾーン紐付け

「系統に追加」操作でSystemNodeを作成し、各ゾーンにsystemIdを設定する。

```typescript
const handleCreateSystem = () => {
  const { createNode, updateNode } = useScene.getState()

  // SystemNode作成
  const systemNode = SystemNode.parse({
    name: `系統-${Date.now()}`,
    servedZoneIds: selectedZoneIds,
  })
  createNode(systemNode, levelId)

  // 各ゾーンにsystemIdを設定
  for (const zoneId of selectedZoneIds) {
    updateNode(zoneId, { systemId: systemNode.id })
  }
}
```

- SystemNode.parse()でZodバリデーション後にcreateNode
- 各ゾーンのsystemIdをupdateNodeで更新
- createNode/updateNodeによりdirtyフラグが自動セットされ、SystemAggregationSystemが再計算

### 4. 🔵 N:1制約バリデーション (REQ-403)

1つのゾーンが複数の系統に属さないようバリデーションを行う。

```typescript
const validateZoneAssignment = (zoneIds: string[]): string[] => {
  const { nodes } = useScene.getState()
  const conflicts: string[] = []

  for (const zoneId of zoneIds) {
    const zone = nodes[zoneId]
    if (zone?.type === 'hvacZone' && zone.systemId) {
      conflicts.push(zoneId)
    }
  }

  return conflicts
}
```

- 既に他の系統に属するゾーンを選択した場合、警告を表示
- ユーザーに「既存系統から外して再割り当て」または「キャンセル」の選択肢を提示
- 同一系統への再追加は無視（冪等性）

### 5. 🔵 グルーピング後のaggregatedLoad即時更新 (REQ-404)

// createNode/updateNodeにより自動的にdirtyフラグがセットされるため、

## 単体テスト要件

### テスト1: SystemNode作成
- **信頼性**: 🔵
- **Given**: 3つのHvacZoneが選択されている
- **When**: 「系統に追加」を実行
- **Then**: SystemNodeが作成され、servedZoneIdsに3つのゾーンIDが含まれる

### テスト2: ゾーンへのsystemId設定
- **信頼性**: 🔵
- **Given**: 3つのHvacZoneが選択されている
- **When**: 「系統に追加」を実行
- **Then**: 各ゾーンのsystemIdが作成されたSystemNodeのIDになる

### テスト3: N:1制約バリデーション — 競合検出
- **信頼性**: 🔵
- **Given**: ゾーンAが既に系統1に属している
- **When**: ゾーンAを系統2に追加しようとする
- **Then**: 競合として検出され、警告が返される

### テスト4: N:1制約バリデーション — 未割当ゾーン
- **信頼性**: 🔵
- **Given**: ゾーンA, B, Cがいずれの系統にも属していない
- **When**: バリデーションを実行
- **Then**: 競合なし（空配列が返される）

### テスト5: 空のゾーン選択
- **信頼性**: 🔵
- **Given**: ゾーンが1つも選択されていない
- **When**: 「系統に追加」を実行
- **Then**: SystemNodeは作成されない（エラーまたは無操作）

### テスト6: 複数選択のトグル動作
- **信頼性**: 🔵
- **Given**: ゾーンA, Bが選択済み
- **When**: Ctrl+クリックでゾーンBを再度クリック
- **Then**: ゾーンBが選択解除され、選択中はゾーンAのみ

- **Toolsルール遵守**: Three.js APIを直接呼ばない、viewerパッケージからインポートしない
